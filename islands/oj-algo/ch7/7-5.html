7.5 贪心策略的选择与更多经典实例

🧭 贪心宗师的试炼：

学到这里，你已经掌握了贪心算法的灵魂。但贪心算法就像是一把没有说明书的“瑞士军刀”。在考场上，没人会告诉你“这道题该用贪心”。你必须自己敏锐地嗅出局部最优的线索，还要小心翼翼地避开那些让你全盘皆输的陷阱！

一、小数背包问题：按单位价值贪心 (7.5.2)

我们在 7.2 节中看到，普通的“0-1背包问题”（宝物不能分割）用贪心会彻底失败。但是，如果这是一道小数背包问题 (Fractional Knapsack) 呢？

规则改变： 宝物不再是坚硬的红宝石，而是可以任意切割的金砂、银粉、铜屑！

💡 绝妙贪心策略：单位价值（性价比）优先！ 算出每种粉末“一公斤值多少钱”，然后疯狂装最贵的那种。如果最后背包装不下了，就把昂贵的粉末切开，只装塞得满缝隙的那一部分！

图解：小数背包的“完美切割”
C++ 小数背包 (性价比排序)
#include <iostream> #include <vector> #include <algorithm> using namespace std; struct Item { double weight; double value; }; // 核心贪心：按 单位价值 (性价比) 从大到小排序 bool cmp(Item a, Item b) { return (a.value / a.weight) > (b.value / b.weight); } double fractionalKnapsack(double capacity, vector<Item>& items) { sort(items.begin(), items.end(), cmp); double totalValue = 0.0; for (auto item : items) { if (capacity >= item.weight) { // 空间充裕，全部装下！ capacity -= item.weight; totalValue += item.value; } else { // 空间不足，切开！按剩余空间乘下单价 totalValue += item.value * (capacity / item.weight); break; // 背包已经装满了 } } return totalValue; }
Python 小数背包 (性价比排序)
def fractional_knapsack(capacity, items): # items 是列表的列表: [[weight, value], ...] # 核心贪心：按 单位价值 (性价比) 从大到小降序排列 items.sort(key=lambda x: x[1] / x[0], reverse=True) total_value = 0.0 for weight, value in items: if capacity >= weight: # 空间充裕，全部装下！ capacity -= weight total_value += value else: # 空间不足，切开！按剩余空间乘下单价 total_value += value * (capacity / weight) break # 背包已经装满了 return total_value # 测试代码 items = [[10, 60], [20, 100], [30, 120]] print(fractional_knapsack(50, items)) # 输出: 240.0

二、最优合并问题：文件/石子合并 (7.5.3)

假设你桌子上有好几堆重量不同的沙子（或者文件），你需要把它们合并成一堆。每次只能把两堆合并在一起，消耗的体力（代价）等于这两堆沙子的重量之和。

注意：如果合并是没有顺序限制的（可以挑任意两堆合并），这就是一道纯粹的贪心题！

💡 贪心策略：永远先合并最小的两堆！ 这和 7.4 节的“哈夫曼树”构造思想完全一模一样！因为越早合并的沙堆，在后续的合并中会被“搬运”越多次。所以我们要让最轻的沙堆被搬运最多次，最重的沙堆最后再搬！

图解：贪心合并（优先队列/最小堆的再次胜利）
C++ 最优合并 (优先队列)
#include <iostream> #include <vector> #include <queue> using namespace std; int minMergeCost(vector<int>& weights) { // 小顶堆，自动将最小的重量浮到最上方 priority_queue<int, vector<int>, greater<int>> minHeap; for (int w : weights) minHeap.push(w); int totalCost = 0; while (minHeap.size() > 1) { // 拿出最轻的两堆 int first = minHeap.top(); minHeap.pop(); int second = minHeap.top(); minHeap.pop(); int cost = first + second; // 计算合并代价 totalCost += cost; // 累加到总体力消耗中 minHeap.push(cost); // 将合并后的新沙堆放回堆中继续参与 } return totalCost; }
Python 最优合并 (heapq 模块)
import heapq def min_merge_cost(weights): # 将列表原地转化为最小堆，只需 O(N) 时间 heapq.heapify(weights) total_cost = 0 while len(weights) > 1: # 拿出最轻的两堆 first = heapq.heappop(weights) second = heapq.heappop(weights) cost = first + second # 计算合并代价 total_cost += cost # 累加到总体力消耗中 heapq.heappush(weights, cost) # 放回新沙堆 return total_cost # 测试代码 print(min_merge_cost([4, 3, 2, 5])) # 输出: 28

三、贪心的局限性与找零反例 (7.5.4 & 7.5.5)

🚨 高危警告：贪心不是万能的！

我们在 7.1 节展示过，如果硬币面值是，每次挑最大的硬币，一定能凑出最少的硬币数量。这种硬币系统在数学上叫做“规范贪心系统”。

但是！一旦国王发了神经，制造了一种面值奇葩的硬币系统（比如去掉了 5 块，换成了 7 块），此时贪心算法不仅会失效，还会显得极其愚蠢！

图解：贪心算法在奇葩硬币系统上的滑铁卢
🐍 Python 语言比拼：短视的贪心 VS 纵观全局的动态规划
❌ 贪心反例：错误示范！
def greedy_fail_example(amount, coins): coins.sort(reverse=True) # 从大到小排 count = 0 for coin in coins: while amount >= coin: amount -= coin count += 1 return count # 目标 14，面值 print(greedy_fail_example(14, [10, 7, 1])) # 贪心输出了 5 (10+1+1+1+1) # 它因为被眼前的10诱惑，错失了最优路线！
✅ 动态规划 (DP)：真正的最优解
def dp_correct_solution(amount, coins): # 备忘录：dp[i]表示凑齐金额 i 需要的最少硬币 # 初始状态填满无穷大 dp = [float('inf')] * (amount + 1) dp[0] = 0 # 凑0元需要0个硬币 for i in range(1, amount + 1): for coin in coins: if i >= coin: # 纵观全局，考虑每一种硬币带来的可能性 dp[i] = min(dp[i], dp[i - coin] + 1) return dp[amount] print(dp_correct_solution(14, [10, 7, 1])) # 完美输出 2 (7+7)！
⚙️ C++ 语言比拼：短视的贪心 VS 纵观全局的动态规划
❌ 贪心反例：错误示范！
int greedyFailExample(int amount, vector<int>& coins) { sort(coins.rbegin(), coins.rend()); // 降序 int count = 0; for (int coin : coins) { while (amount >= coin) { amount -= coin; count++; } } return count; } // 目标 14，面值 // cout << greedyFailExample(14, coins); // 输出 5，这是错的！
✅ 动态规划 (DP)：真正的最优解
int dpCorrectSolution(int amount, vector<int>& coins) { // 使用 amount + 1 作为“无穷大”的安全替代值 vector<int> dp(amount + 1, amount + 1); dp[0] = 0; for (int i = 1; i <= amount; i++) { for (int coin : coins) { if (i >= coin) { // 状态转移：保留使用这枚硬币与不使用的较小值 dp[i] = min(dp[i], dp[i - coin] + 1); } } } return dp[amount] > amount ? -1 : dp[amount]; } // cout << dpCorrectSolution(14, coins); // 完美输出 2！

🎉 贪心章节结语：如何选择魔法？

贪心算法就像是一场豪赌，它代码简短、执行速度极快（通常是 O(N) 或 O(N log N)）。

如果题目允许对物品进行切割（小数背包）、允许随意排序合并（哈夫曼、区间调度），那么大胆使用贪心吧！

一旦你发现题目有“硬性绑定”、不能切割（0-1背包），或者出现“拿了眼前的最大值，反而堵死了后面一条坦途”的情况（奇葩硬币找零）。此时就说明该问题不具备贪心选择性质。深吸一口气，准备翻开下一章的终极魔法——动态规划 (Dynamic Programming)，它会帮你遍历所有的可能性，找到真正的宇宙最优解！
