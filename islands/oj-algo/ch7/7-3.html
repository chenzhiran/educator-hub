7.3 区间问题 (Interval Problems)

📏 时间与空间的博弈：

在算法考试和面试中，有一类超级经典的题目，它们总是给你一大堆“线段”或者“时间段”（也就是区间），这些区间往往会互相重叠。然后让你找出某种最优的安排方式。

这类问题简直就是为贪心算法量身定制的！但陷阱在于：面对乱糟糟的区间，我们第一步到底该按“开始时间”排序，还是按“结束时间”排序？ 选错标准，就会满盘皆输！

一、区间调度 (活动选择问题) (7.3.1)

🎯 目标： 你是一个大忙人，一天内有很多活动可以参加（都有确定的开始和结束时间）。但你分身乏术，同一时间只能参加一个。如何安排才能参加最多数量的活动？

💡 贪心策略：最早结束优先！ 每次都在剩下的活动中，挑选“结束时间最早”的。因为它早早结束了，就能把更多的时间留给后面的活动！

图解：区间调度（尽早脱身，才能参加下一场）
C++ 区间调度
#include <iostream> #include <vector> #include <algorithm> using namespace std; struct Interval { int start, end; }; // 核心策略：按结束时间从小到大排序 bool cmp(Interval a, Interval b) { return a.end < b.end; } int activitySelection(vector<Interval>& itvs) { if (itvs.empty()) return 0; sort(itvs.begin(), itvs.end(), cmp); int count = 1; // 第一场活动必然参加 int currentEnd = itvs[0].end; for (int i = 1; i < itvs.size(); i++) { // 如果下一场活动的开始时间 >= 当前结束时间 if (itvs[i].start >= currentEnd) { count++; currentEnd = itvs[i].end; // 更新防冲突的时间线 } } return count; }
Python 区间调度
def activity_selection(intervals): if not intervals: return 0 # 核心策略：按结束时间从小到大排序 (x 代表结束时间) intervals.sort(key=lambda x: x[1]) count = 1 # 第一场活动必然参加 current_end = intervals[0][1] for i in range(1, len(intervals)): # 如果下一场活动的开始时间 >= 当前结束时间 if intervals[i][0] >= current_end: count += 1 current_end = intervals[i][1] # 更新时间线 return count # 示例调用 itvs = [[1,5], [0,3], [4,6], [7,10]] print(activity_selection(itvs)) # 输出: 3

二、区间划分 (最少教室问题) (7.3.2)

🎯 目标： 学校有 N 门课，时间和上面一样可能冲突。你必须安排所有课程。最少需要准备多少间教室，才能保证没有课程冲突？

💡 贪心策略：最早开始优先！ 把课程按开始时间排好队，一节一节来。如果当前有空闲的教室，就放进去（更新该教室结束时间）；如果所有教室都在上课，就只能新开一间教室。

图解：区间划分（教室不够，只能扩建）

🛠️ 代码实现提示：

这里往往需要使用一个优先队列（最小堆）来维护所有正在使用的教室的“最早结束时间”。来一门新课，就去堆顶看看最快下课的那间教室空出来了没。

C++ 最少教室
#include <iostream> #include <vector> #include <algorithm> #include <queue> using namespace std; struct Interval { int start, end; }; // 按开始时间排序！ bool cmp(Interval a, Interval b) { return a.start < b.start; } int minClassrooms(vector<Interval>& itvs) { if (itvs.empty()) return 0; sort(itvs.begin(), itvs.end(), cmp); // 小顶堆，存储每个教室的结束时间 priority_queue<int, vector<int>, greater<int>> minHeap; for (auto i : itvs) { // 如果堆不为空，且新课开始时间 >= 最早下课的教室时间 if (!minHeap.empty() && minHeap.top() <= i.start) { minHeap.pop(); // 腾出这间旧教室 } // 无论如何，存入这节课的结束时间 // (要么占用旧教室，要么开辟新教室) minHeap.push(i.end); } // 堆里剩下多少个元素，就说明开了多少间教室 return minHeap.size(); }
Python 最少教室
import heapq def min_classrooms(intervals): if not intervals: return 0 # 按开始时间排序！ intervals.sort(key=lambda x: x[0]) # 小顶堆，存储每个教室的结束时间 heap = [] for itv in intervals: # 若堆不为空，且新课开始 >= 最早下课时间 if heap and heap[0] <= itv[0]: heapq.heappop(heap) # 腾出这间教室 # 将这节课的结束时间压入堆 heapq.heappush(heap, itv[1]) # 堆的大小即为最少教室数 return len(heap)

三、区间覆盖 (7.3.3)

🎯 目标： 给定一个大线段（比如从坐标 1 到坐标 8），还有一堆零碎的小线段。如何用最少的小线段，把这个大线段完整地覆盖住？

💡 贪心策略：尽力向右延伸！ 按开始时间排序。在所有能覆盖到当前起点的小线段中，挑选那个右端点跑得最远的。这样我们每次都能贪心地获得最大的覆盖范围！

图解：区间覆盖（每次都挑选能延伸最远的接力棒）
C++ 区间覆盖
// 此处代码做简化演示，重点看逻辑结构 int minIntervalCover(vector<Interval>& itvs, int start, int end) { sort(itvs.begin(), itvs.end(), cmpByStart); // 按起点排序 int res = 0, i = 0, n = itvs.size(); while (start < end) { int maxRight = -1; // 遍历所有能覆盖当前 start 的区间，找最远的 while (i < n && itvs[i].start <= start) { maxRight = max(maxRight, itvs[i].end); i++; } // 如果连当前起点都覆盖不到，说明有断层，失败！ if (maxRight < start) return -1; // 贪心选择成功，更新新的覆盖起点 res++; start = maxRight; } return res; }
Python 区间覆盖
def min_interval_cover(intervals, target_start, target_end): # 按开始时间排序 intervals.sort(key=lambda x: x[0]) res = 0 i = 0 n = len(intervals) while target_start < target_end: max_right = -float('inf') # 找能覆盖当前起点且右端点最远的区间 while i < n and intervals[i][0] <= target_start: max_right = max(max_right, intervals[i][1]) i += 1 if max_right < target_start: return -1 # 断层了，无法覆盖 res += 1 target_start = max_right # 更新起点 return res

四、区间选点问题 (7.3.4)

🎯 目标： 有很多气球水平漂浮在不同的区间范围。你站在下面往上射箭，射出的箭是一条垂直线。最少射多少箭，才能把所有气球都扎破？

💡 贪心策略：尽拖延，射尾巴！ 这和第一节的“区间调度”非常像。也是按结束时间排序。我们尽量把射箭的位置拖到“当前第一个要消失的气球”的尾部。这样一箭上去，顺带扎破的其他气球肯定是最多的！

图解：射气球（区间选点），箭射在结束边界收益最大！

五、各问题贪心策略大盘点 (7.3.5)

做区间题目，千万不能背代码，要背**“排序规则”**！请牢记下面这张“寻宝秘籍地图”：

问题类型 排序依据 🌟 贪心策略核心思想
区间调度 (最多活动) 按 结束时间 升序 尽量早点结束，给后面留出更多空闲时间。
区间选点 (射气球) 按 结束时间 升序 射箭位置拖到最后（结束点），顺带穿透最多的重叠区间。代码和调度几乎一样！
区间划分 (最少教室) 按 开始时间 升序 顺着时间流发展。早开始的先安排，用小顶堆管理空余教室。
区间覆盖 (拼大线段) 按 开始时间 升序 在能衔接起点的前提下，挑选能往右延伸最远的区间，步子迈得越大越好。
