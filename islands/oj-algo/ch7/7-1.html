7.1 贪心算法的基本思想

💰 海盗与红宝石：

如果面前有一大堆红宝石，你的背包只能装 5 颗，你会怎么选？显然，你会每次都从堆里挑出当前最大的那一颗，挑满 5 次为止！

这就是贪心算法的灵魂：不纠结复杂的未来，只看眼前，每次都做出在当前看来最好的选择！

一、贪心选择性质与最优子结构 (7.1.1)

不是所有的问题都能用“贪心”来解决。要想用贪心算法找到真正的宝藏，这个问题必须具备两张“通行证”：

1. 贪心选择性质 (Greedy-choice Property)

这指的是：我们可以通过做出局部最优的选择（也就是只顾眼前的最优选），来最终达到全局最优解。在做贪心选择时，我们绝不依赖未来可能发生的事情，也绝不反悔（不回溯）。

2. 最优子结构 (Optimal Substructure)

这指的是：当我们做完一次贪心选择后，原问题变成了一个规模更小的“子问题”。如果原问题的最优解，正好包含了这个子问题的最优解，我们就说它具备最优子结构。

二、贪心算法的一般步骤与代码实现 (7.1.2)

在使用贪心魔法编写程序时，通常需要经历以下四个核心步骤：

第一步： 把需要解决的问题，转化成数学模型（明确目标是什么，约束条件是什么）。

第二步： 把大问题分解为一连串的小问题。

第三步： 为每一个小问题寻找一个“贪心策略”（比如每次都选面值最大的，或者每次都选重量最轻的）。

第四步： 编写循环或递归代码，将每一次的局部最优解收集起来，合并为最终的答案。

🛠️ 经典实战：找零钱问题

假设我们有面值分别为 25元、10元、5元、1元的硬币。现在需要找给客人 36 元。怎样找零才能使硬币的数量最少？

贪心策略： 每次都尽量使用面值最大的硬币，直到凑齐金额。

C++ 贪心找零
#include <iostream> #include <vector> using namespace std; void greedyCoinChange(int amount) { // 硬币面值（必须提前从大到小排好序） int coins[] = {25, 10, 5, 1}; int count = 0; cout << "使用的硬币: "; // 遍历每一种面值的硬币 for (int i = 0; i < 4; i++) { // 只要当前剩余金额足够大，就一直用这种硬币 while (amount >= coins[i]) { amount -= coins[i]; cout << coins[i] << " "; count++; } } cout << "\n总共硬币数: " << count << endl; } int main() { greedyCoinChange(36); return 0; }
Python 贪心找零
def greedy_coin_change(amount): # 硬币面值（必须提前从大到小排好序） coins = [25, 10, 5, 1] count = 0 result = [] # 遍历每一种面值的硬币 for coin in coins: # 只要当前剩余金额足够大，就一直用这种硬币 while amount >= coin: amount -= coin result.append(coin) count += 1 print(f"使用的硬币: {result}") print(f"总共硬币数: {count}") # 测试代码 greedy_coin_change(36)

三、贪心与动态规划的对比 (7.1.3)

很多同学在学完“动态规划（DP）”后，经常会把它和“贪心”搞混。它们都用来求最优解，都要求最优子结构。但它们的策略截然不同：

🏃‍♂️ 贪心算法（只看眼前，一往无前）：
自顶向下，每一步都做出当前看起来最棒的选择，选完之后绝不回头重新考虑。速度极快，但如果问题不符合“贪心选择性质”，它找到的可能只是个局部最优解，而不是真正的全局最优解。

🧠 动态规划（深思熟虑，统揽全局）：
通常是自底向上，把所有可能的子问题的答案都计算并保存下来（依靠备忘录或DP表）。在做决策时，它会回头查看所有过去的历史记录，挑选出一条能够导致最终结果最优的路线。速度较慢，但绝不出错。
