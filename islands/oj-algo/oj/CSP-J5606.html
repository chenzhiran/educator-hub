在这里，我们需要吟唱一串神秘的“能量序列”。 序列会由一个正整数 n 开启，每次我们可以在末尾添加一个新数字，但是这个新数字绝对不能超过“上一个数字的一半”！ 这样分岔交错的小路，一共有多少条不同的“秘径”可以成功通往山脚呢？ 让我们用递归与动态规划的智慧来解开它！

📜 任务清单 (题目描述)

给定一个正整数 n，我们需要构造一个满足下列规则的正整数序列：

能量序列的构筑法则：

  • 序列的第一个数字必须是 n。
  • 在序列的队尾可以随时接一个新的正整数，但是这个新数字不能超过当前最后一个数字的一半。
  • 一直遵守这个规则，直到再也无法或不想继续往后接新数字为止，这样得到的每一条序列都是一条合法的“秘径”。

判定不同的方式： 只要两条序列的长度不同，或者任何一个位置上的数字不同，就视为两条不同的秘径。 请问从 n 出发，总共能构造出多少条不同的合法序列？

📥 魔法输入阵：
(一个正整数 n，1 ≤ n ≤ 1000)
6
📤 奇迹输出阵：
(一个整数，代表可行的秘径总数)
6

样例解释说明： 当 n=6 时，合法的 6 条秘径序列分别为：
1. （直接结束）
2. （1 不超过 6 的一半）
3. （2 不超过 6 的一半）
4. （1 不超过 2 的一半）
5. （3 不超过 6 的一半）
6. （1 不超过 3 的一半）

🧠 魔法道具箱 (核心图解：树状分岔与两种解法)

要解决这个问题，我们可以从递归的基本逻辑出发，然后通过高阶的线性优化法术来实现全屏秒杀！

🌲 解法 1：记忆化递归（自顶向下的分岔树） 🌲

假设我们要计算从数字 x 出发能构成的合法序列条数 f(x)。 根据规则，它可以分为以下两部分：

1. 只包含它本身一根独苗的序列（贡献为 1 条）。
2. 在它的后面接上一个新的数字 i，而这个 i 只能从 1 挑选到 x/2（向下取整）。 接上 i 之后，就相当于变成了“从 i 出发的秘径”！

数量：1

[6, 3...]
数量：f(3)
💡 递归转移公式：
f(x) = 1 + f(1) + f(2) + ... + f(x / 2)

为了防止递归反复计算相同的数超时，我们可以建立一个记忆本数组，这就是带记忆化的递归，时间复杂度是 O(n^2)。

⚡ 解法 2：线性递推优化（神级 O(n) 降维打击） ⚡

让我们静下心来，观察两个相邻的数字，比如 5 和 6：

根据公式：
f(5) = 1 + f(1) + f(2) （5 的一半向下取整是 2）
f(6) = 1 + f(1) + f(2) + f(3) （6 的一半向下取整是 3）

看到了吗！f(6) 和 f(5) 相比，前面的项一模一样，唯独最后多出了一项 f(3)！ 这里的 3 恰好就是 6 / 2。

x 是奇数（如 5）
一半的限制不变
➡️ f(x) = f(x - 1)
✨
x 是偶数（如 6）
刚好解锁了接上 x/2 这一支
➡️ f(x) = f(x - 1) + f(x / 2)

通过这个惊天的大发现，我们可以彻底扔掉内层的 for 循环求和，直接用 O(1) 的时间从前一项推出后一项！ 整个问题的总时间复杂度被降级成了极其恐怖的 O(n)！

📘 C++ 魔法秘籍 (知识点)

  • ⚙️ 记忆化搜索的机制： 在第一个程序（递归版）中，我们开辟了全局的 memo 动态数组并初始化为 -1。 每次进入函数先进行检查，如果算过则瞬间返回，这体现了自顶向下的分治记忆思想。
  • 🛡️ 线性递推与 long long： 虽然这道题的 n 只有 1000，但在做类似计数或阶乘相关的递归题目时，结果可能呈爆炸式增长。 题目提示我们为了安全起见，一定要使用 64 位整型 long long 来存储答案。

💻 C++ 终极法术 1 (记忆化递归版：契合递归练习)

#include <iostream>
#include <vector>
using namespace std;

// 📖 魔法记忆本：专门阻止递归陷入重复计算的深渊
vector<long long> memo;

// 🎨 递归计算从正整数 x 出发能构成的秘径序列条数
long long f(int x) {
    if (x == 1) return 1; // 🏁 边界条件：1 的后面什么都接不了，只有 自身这1条
    
    // 🛡️ 记忆化查表：如果之前曾经施法计算过，直接翻书拿答案
    if (memo[x] != -1) return memo[x];
    
    long long res = 1; // ✨ 首先包含只由自己组成的单数序列
    
    // 🔍 遍历所有合法的后继数字 i (不能超过当前数字的一半)
    for (int i = 1; i <= x / 2; ++i) {
        res += f(i); // 累加每一个分岔子树带来的序列数
    }
    
    memo[x] = res; // ✍️ 将最优解记入小本本
    return res;
}

int main() {
    int n;
    cin >> n;
    
    // 🧹 将记忆本容量扩展为 n+1，全部涂成 -1 代表尚未算过
    memo.assign(n + 1, -1);
    
    cout << f(n) << endl;
    return 0;
}

💻 C++ 终极法术 2 (极致优化 O(n) 线性递推版)

#include <iostream>
#include <vector>
using namespace std;

int main() {
    int n;
    cin >> n;
    
    vector<long long> f(n + 1);
    f[1] = 1; // 🎨 初始阵型基石
    
    // ⚡ 一层 for 循环，利用奇偶数规律直接推出全表
    for (int x = 2; x <= n; ++x) {
        // 如果是偶数，多解锁了接上 x/2 的能力；如果是奇数，方案数跟前一项完全相同
        f[x] = (x % 2 == 0) ? f[x - 1] + f[x / 2] : f[x - 1];
    }
    
    cout << f[n] << '\n';
    return 0;
}

🐍 Python 魔法秘籍 (知识点)

  • ✨ 内置黑魔法 @lru_cache： 在 Python 递归版代码中，我们直接引入了工具箱里的 from functools import lru_cache。这是一个极其强大的内置缓存装饰器，只要在自定义函数上方随手写一行 @lru_cache(maxsize=None)，Python 就会在幕后默默帮我们把所有的函数输入与对应的返回值存进字典里，实现全自动的记忆化搜索，代码极其高雅简洁！
  • 🚀 整除切分符 //： 请记住，Python 的单斜杠 / 算出的是浮点数（小数），如果想要取得满足规则的、向下取整的“一半”整数，必须使用双斜杠整除符号 //。

💻 Python 终极法术 1 (记忆化递归版)

from functools import lru_cache
import sys

# 🧙‍♂️ 施加全自动查表魔法，免去我们手动写字典的烦恼
@lru_cache(maxsize=None)
def f(x):
    if x == 1:
        return 1  // 🏁 1 后面无法追加任何数字，只有 一种可能
        
    total = 1  # ✨ 首先包括只含有一个数 x 自身的单元素序列
    
    # 🔍 寻找所有不超过上一个数字一半的后继整数 i
    for i in range(1, x // 2 + 1):
        total += f(i)  # 递归呼叫子问题并累加
        
    return total

def main():
    # 📥 接入雪山祭坛的原始能量
    n = int(sys.stdin.read().strip())
    print(f(n))

if __name__ == "__main__":
    main()

💻 Python 终极法术 2 (极致优化 O(n) 线性递推版)

import sys

def main():
    n = int(sys.stdin.read().strip())
    
    # 📖 准备一维递推账本
    f = [0] * (n + 1)
    f[1] = 1
    
    # ⚡ 自底向上单层循环，光速递推
    for x in range(2, n + 1):
        if x % 2 == 0:
            f[x] = f[x - 1] + f[x // 2] # 偶数时解锁新分岔
        else:
            f[x] = f[x - 1]             # 奇数时与前一项相同
            
    print(f[n])

if __name__ == "__main__":
    main()

💡 技巧和重点详细总结 (一定要看)

  • 🌟 递归到递推的底层相通： 很多初学者觉得“递归”和“用 for 循环动态规划递推”是两个互不相干的法术。实际上，带记忆化的递归（自顶向下）和循环递推（自底向上）在数学的本质结构上是完全同一对孪生兄弟！ 掌握了用小笔记本记录答案的递归，你就已经领悟了动态规划的真谛。
  • 🌟 发现隐藏的局部规律： 面对一个复杂的求和递推式，千万不要急着直接写双重循环开跑。试着在纸上把前几项（如 f(5)、f(6)）展开并进行**差分对比**，你会经常发现相邻两项之间庞大的求和公式可以被神奇地合并简化。 像本题中利用奇偶性把 O(n^2) 降维打击成 O(n)，就是算法优化中极具美感的操作！
  • 🌟 边界条件的牢固性： 写任何递归和递推代码，基础的边界条件（Base Case）是让魔法停下来的最重要锚点。 比如这道题，f(1) = 1 是全场迭代的根基，失去了这个 1，所有的累加结果都将变成 0，做题时千万要先把边界牢牢锁死！
