📜 任务清单 (题目描述)

给定一个整数 n，求 n 个元素的错排个数。

错排的定义：

  • 把 1 到 n 这 n 个数字重新排列。
  • 规定第 i 个数字绝对不能放在第 i 个位置上（即每个人都走错房间）。

计算任务： 请问一共有多少种合法的错排方式？

📥 魔法输入阵：
(一个整数 n，代表药水瓶的数量)
4
📤 奇迹输出阵：
(一个整数，代表合法的错排数量)
9

🧠 魔法道具箱 (核心图解：递归降维打击)

这道题是练习递归思想的绝佳素材！假设我们用 f(n) 表示 n 个元素的错排数量。现在，我们要把第 n 号药水瓶放进别的盒子里，看看会发生什么连锁反应：

🧪 第 n 号瓶的命运抉择 🧪

首先，n 号瓶绝对不能放在 n 号盒子里。所以，它可以随便挑一个前面的盒子 k。一共有 (n-1) 个盒子可以挑。

一旦 n 号瓶霸占了 k 号盒子，那原本属于 k 号盒子的 k 号瓶该怎么办呢？它有两种选择：

  • 情况 1：和平互换
    n号瓶
    ⬇️跑去⬇️
    k号盒
    🤝
    k号瓶
    ⬇️跑去⬇️
    n号盒
    k 号瓶说：“既然你抢了我的盒子，那我就去你的 n 号盒子吧！”
    这样一来，n 号瓶和 k 号瓶完美互换，它们俩的问题彻底解决了！剩下的 (n-2) 个瓶子继续它们自己的错排游戏。
    👉 这转化为规模更小的子问题：f(n-2)
  • 情况 2：不妥协的流浪
    n号瓶
    ⬇️跑去⬇️
    k号盒
    ❌ 不去 n 号盒！
    k号瓶
    ⬇️跑去⬇️
    其他盒子
    k 号瓶说：“我才不去你的 n 号盒子呢！”
    此时 n 号盒子还是空的，并且 k 号瓶拒绝进入它。这简直就像是 k 号瓶继承了 n 号瓶的遗志——它有了一个不能去的“禁区”。除了已经安置好的 n 号瓶，场上还剩下 (n-1) 个瓶子和 (n-1) 个盒子，且每个瓶子都刚好有一个不能去的盒子。这在逻辑上完全等价于 (n-1) 个元素的错排！
    👉 这转化为另一个规模的子问题：f(n-1)
💡 终极递归推导公式：
f(n) = (n - 1) × ( f(n-1) + f(n-2) )

基础终止条件：f(1) = 0（1个人肯定走不错），f(2) = 1（2个人只能互相交换）。

📘 C++ 魔法秘籍 (知识点)

  • ⚙️ 带记忆的递归 (Memoization)： 纯粹的递归会因为反复计算相同的分支而变得非常慢（指数级时间）。我们开辟一个 memo 数组，把算过的 f(n) 存起来，下次再问到它，直接查表返回，瞬间将时间复杂度降为 O(n)！
  • 🛡️ 类型的选择： 即便 n 只到 20，错排的数量增长也是极其可怕的！仔细看代码，必须要使用 long long 才能装下这巨大的天文数字哦。

💻 C++ 终极法术 (记忆化递归版)

#include <iostream>
using namespace std;

// 📖 魔法记忆笔记本：专门记录已经算过的错排数
long long memo[25] = {0};

// 🌟 核心递归函数 f(n)
long long f(int n) {
    // 🏁 递归终止条件
    if (n == 1) return 0; // 1 个瓶子没法错排
    if (n == 2) return 1; // 2 个瓶子只有 1 种换法
    
    // 🛡️ 如果笔记本里已经记过这个 n 的答案，直接拿出来用！(记忆化)
    if (memo[n] != 0) {
        return memo[n];
    }
    
    // 🧙‍♂️ 施放法术：(n-1) 个选择 × (和平互换 f(n-2) + 不妥协 f(n-1))
    memo[n] = (n - 1) * (f(n - 1) + f(n - 2));
    
    return memo[n];
}

int main() {
    int n;
    cin >> n;
    
    // 📤 呼叫函数并输出最终答案
    cout << f(n) << endl;
    
    return 0;
}

🐍 Python 魔法秘籍 (知识点)

  • ✨ 字典缓存大法： 在 Python 中，实现记忆化递归最舒服的方式就是创建一个全局字典 memo = {}。通过 if n in memo: 就能轻松避免重复计算。
  • 🚀 从递归到递推： 既然 f(n) 只和 f(n-1) 以及 f(n-2) 有关，很多老练的魔法师会选择只用三个变量 a, b, c 来滚动计算（这叫做递推或者动态规划），这样连字典都不用建，速度更快哦！不过为了练习递归思维，我们这里的代码依然呈现原汁原味的递归风味。

💻 Python 终极法术 (记忆化递归版)

import sys

# 📖 魔法记忆笔记本：用字典来记录算过的答案
memo = {}

# 🌟 核心递归函数
def f(n):
    # 🏁 递归终止条件
    if n == 1:
        return 0
    if n == 2:
        return 1
        
    # 🛡️ 记忆化查表：算过了就直接返回，绝不加班
    if n in memo:
        return memo[n]
        
    # 🧙‍♂️ 施放法术：(n-1) 个选择 × (和平互换 f(n-2) + 不妥协 f(n-1))
    memo[n] = (n - 1) * (f(n - 1) + f(n - 2))
    
    return memo[n]

def main():
    # 📥 读取想要计算的瓶子数量
    input_data = sys.stdin.read().split()
    if not input_data:
        return
    n = int(input_data[0])
    
    # 🏆 施放法术，打印出最终的错排总数
    print(f(n))

if __name__ == "__main__":
    main()

💡 技巧和重点详细总结 (一定要看)

  • 🌟 递归分解的艺术： “错排”可以说是高中数学和算法中最经典的排列组合问题。初看时觉得情况太乱根本数不清，但是只要我们盯住最后一个元素，并对它占领的位置的原主人（k号）做分类讨论（去还是不去n号），就能瞬间把庞大的乱麻切分成两个已知的子问题！这是算法中最顶级的降维打击思维！
  • 🌟 记忆化（Memoization）不可或缺： 在写类似于斐波那契数列、错排公式这种 f(n) 依赖于前几项的递归时，必须加数组/字典进行缓存。否则当你计算 f(20) 时，它会展开成上百万个树枝，计算到天荒地老！加了记忆化之后，它就只算 20 次，速度堪比光速。
  • 🌟 动态规划的基石： 掌握了带记忆的递归之后，你其实半只脚已经踏入了“动态规划（DP）”的大门。像题目原描述里给出的那段 C++ 代码，就是省略了函数调用，直接用一个 for 循环从 3 算到 n，那是递归算法修炼到极致后的最终形态（自底向上递推），但是它们的内在数学灵魂是完全一模一样的！
