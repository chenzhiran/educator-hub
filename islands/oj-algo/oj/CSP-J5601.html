📜 任务清单 (题目描述)

在四座塔（A、B、C、D）的情况下，我们需要将 A 座上的 n 个圆盘移动到 D 座上。

神圣的搬运法则：

  • 每次只能移动一个圆盘。
  • 移动过程中必须始终保持小盘在上，大盘在下。

计算任务： 请问至少要移动多少步才能完成任务？

📥 魔法输入阵：
(圆盘的数量)
5
📤 奇迹输出阵：
(需要的最少步数)
13

🧠 魔法道具箱 (核心图解：四柱的奥秘)

普通的汉诺塔只有三根柱子，而四根柱子给我们带来了新的可能！我们需要将问题拆解开来：

🏗️ 四柱汉诺塔的完美三步曲 🏗️
  • 步骤 1 先把 A 柱顶端的 j 个较小的盘子，在 4 根柱子的帮助下，移到某根空柱（比如 B 柱）上。
    消耗步数：h4[j]
  • 步骤 2 现在 B 柱被那 j 个盘子占用了！A 柱还剩 n-j 个大盘子。我们只能用剩下的 3 根柱子（A、C、D）把这 n-j 个大盘子移到目标柱 D 柱上。
    消耗步数：退化为三柱汉诺塔 h3[n-j]
  • 步骤 3 最后，再把 B 柱上的 j 个小盘子，在 4 根柱子的帮助下，移到 D 柱上。
    消耗步数：h4[j]

✨ 状态转移与极限求极小值

我们不知道把多少个盘子（也就是 j 等于多少）先移走才是最划算的，所以我们可以遍历所有的 j（从 1 到 n-1），找出总步数最小的那种切分方案！

💡 经典三柱汉诺塔公式：
$$h_3(n) = 2 \times h_3(n-1) + 1$$
💡 四柱汉诺塔动态规划公式：
$$h_4(n) = \min_{1 \le j < n} \{ 2 \times h_4(j) + h_3(n-j) \}$$

📘 C++ 魔法秘籍 (知识点)

  • ⚙️ 记忆化搜索 (Memoization)： 递归虽然好理解，但是如果不用数组把算过的答案存起来，它会傻乎乎地重复计算，导致“超时”！代码里的 h3 和 h4 数组就是我们的“记忆笔记本”。
  • 🛡️ 初始化 -1： 使用 memset(h3, -1, sizeof h3) 把笔记本初始化为 -1，代表“还没算过”。当看到值不是 -1 时，直接把答案拿出来用即可，速度起飞！

💻 C++ 终极法术 (代码详解)

#include <bits/stdc++.h>
using namespace std;

const int N = 20000;
int h3[N], h4[N]; // 魔法笔记本，用于记录计算结果

// 🎨 计算 3 根柱子时移动 i 个盘子的最少步数
int calc_h3(int i) {
    if (i == 1) return 1; // 🏁 基本情况
    if (h3[i] != -1) return h3[i]; // 📖 如果算过了，直接翻笔记本
    
    // 经典汉诺塔递推公式：移走上方 i-1 个，移底盘，再移回上方 i-1 个
    return h3[i] = 2 * calc_h3(i - 1) + 1;
}

// 🎨 计算 4 根柱子时移动 i 个盘子的最少步数
int calc_h4(int i) {
    if (i == 1) return 1; // 🏁 基本情况
    if (h4[i] != -1) return h4[i]; // 📖 查阅笔记本

    int result = INT_MAX; // 初始设为无穷大，为了寻找最小值
    // 🔍 遍历所有的分割方案，寻找最完美的切分点 j
    for (int j = 1; j < i; ++j) {
        result = min(result, 2 * calc_h4(j) + calc_h3(i - j));
    }
    
    return h4[i] = result; // 把算出的最优解写进笔记本
}

int main() {
    int n;
    cin >> n;

    // 🧹 初始化数组为 -1，清除笔记本
    memset(h3, -1, sizeof h3);
    memset(h4, -1, sizeof h4);

    cout << calc_h4(n) << endl;

    return 0;
}

🐍 Python 魔法秘籍 (知识点)

  • ✨ 字典作记忆本： Python 中我们可以直接使用字典（Dictionary）或者内置的 @lru_cache 装饰器来实现记忆化。这里我们用字典来模拟 C++ 中的数组记录过程。
  • ⚠️ 递归深度限制： 汉诺塔的步数和层数增加很快，为了防止神殿法力崩溃（RecursionError），我们需要在代码开头用 sys.setrecursionlimit() 解除递归深度封印。

💻 Python 终极法术 (代码详解)

import sys
# 🛡️ 提升魔法上限，防止递归层数过深导致报错
sys.setrecursionlimit(20000)

# 📖 魔法笔记本，用于记录算过的结果，拒绝重复劳动
memo_h3 = {}
memo_h4 = {}

# 🎨 计算 3 根柱子的情况
def calc_h3(i):
    if i == 1:
        return 1
    if i in memo_h3:
        return memo_h3[i]
        
    memo_h3[i] = 2 * calc_h3(i - 1) + 1
    return memo_h3[i]

# 🎨 计算 4 根柱子的情况
def calc_h4(i):
    if i == 1:
        return 1
    if i in memo_h4:
        return memo_h4[i]

    min_steps = float('inf') # 设为无穷大
    # 🔍 遍历 1 到 i-1 所有的分割方案 j，寻找最小值
    for j in range(1, i):
        min_steps = min(min_steps, 2 * calc_h4(j) + calc_h3(i - j))

    memo_h4[i] = min_steps
    return memo_h4[i]

def main():
    n = int(input())
    print(calc_h4(n))

if __name__ == "__main__":
    main()

💡 技巧和重点总结 (划重点啦)

  • 🌟 降维打击的核心思想： 遇到四柱汉诺塔不会做？我们就把它拆分成我们会做的三柱汉诺塔！通过让出一部分柱子，把复杂问题转化为已知问题。
  • 🌟 寻找最优解 (DP思想的萌芽)： 我们并不知道切分点在哪，既然不知道，那就让程序用一个 for 循环试遍所有的可能性，然后用 min() 函数选出步数最少的那一个。
  • 🌟 记忆化搜索： 这种“画个树叉”一直递归下去的方法，往往有很多分支计算的内容是完全一模一样的。使用数组或字典记录下算过的结果，是把时间复杂度从指数级降下来的关键法宝！
