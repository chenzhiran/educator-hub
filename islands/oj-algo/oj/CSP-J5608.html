📜 任务清单 (题目描述)

在 n 乘 n 的魔法棋盘上放下 n 枚“冰后”（每行恰好一枚）。

冰后的绝不冲突法则：

  • 每一列也必须恰好只有一枚冰后。
  • 任意一条对角线（包含两条主对角线及其所有平行倾斜线）上，至多只能存在一枚冰后。

我们可以用一个数字序列来记录阵法：第 i 个数字表示第 i 行皇后所在的列号。例如，在 6 乘 6 的棋盘中，一种完美阵型可以描述为：2 4 6 1 3 5。

计算任务： 给定棋盘尺寸 n，请按照字典序输出前三个合法阵型。最后一行输出该棋盘尺寸下，所有合法解的总数量。

🧠 魔法道具箱 (核心图解：对角线常量的秘密)

经典的 N 皇后问题！我们采用回溯搜索（DFS）：按行依次放下冰后。只要解决了“怎么用最快速度判断对角线冲突”，就能无伤通关！

🔍 冰雪极速冲突判定术：

  • 1. 直线列冲突 用一个简单的布尔数组 col[c] 来标记，如果第 c 列被放了冰后，就设为 true。
  • 2. 主对角线（左上到右下） 观察规律可以惊奇地发现：在同一条主对角线上，所有格子的“行号减列号 (row - c)”结果竟然是一个完全一样的常数！由于相减可能出现负数，我们统一加上棋盘大小 n 作为偏移，即使用 row - c + n 作为数组下标来唯一标识一条主对角线。
  • 3. 副对角线（右上到左下） 同理，在同一条副对角线上，所有格子的“行号加列号 (row + c)”结果也是一个一模一样的常量！直接用 row + c 作为下标即可完美判定冲突！
💡 阵法无冲突判定公式：
如果 !col[c] 且 !d1[row - c + n] 且 !d2[row + c] 成立，则当前位置安全，可放置冰后！

要保持天然的字典序，我们在遍历列号 c 时，必须从 1 到 n 从小到大进行尝试。

📘 C++ 魔法秘籍 (知识点)

  • ⚙️ 回溯搜索的基本结构： 尝试放置第 row 行的冰后 ➡️ 成功找到一个安全列 c ➡️ 标记占用，将 c 加入结果 ➡️ 递归进入下一行 dfs(row + 1) ➡️ 关键一步：恢复现场 (Backtracking)，把标记清除，弹出 c，继续尝试下一个位置！
  • 🛡️ 结果的拦截： 当 row == n 时说明成功发现了一个完整阵型，总计数加 1。因为只需要前三个解，所以当 firstThree.size() < 3 时，才把当前方案塞进记录本。

💻 C++ 终极法术 (清晰直观回溯版)

#include <iostream>
#include <vector>
using namespace std;

int n;
vector<int> cur;                  // 收集当前正在推演的解
vector<vector<int>> firstThree;   // 仅储存前 3 个满分法阵
int total = 0;                    // 全局累加器，统计总方案数

// 🔮 三个真假布尔阵：列占用、主对角线占用、副对角线占用
bool col[15], d1[30], d2[30]; 

// 🎨 核心深度优先搜索回溯函数
void dfs(int row) {
    // 🏁 终止条件：当 row 从 0 一直顺利走到 n-1 后，row 变成 n 说明全部行都放好了！
    if (row == n) {
        total++; // 发现了一条全新的秘径
        if (firstThree.size() < 3) {
            firstThree.push_back(cur); // 只有前 3 个解能入选展示集
        }
        return;
    }
    
    // 🔍 按照列号从小到大升序遍历，这保证了最先搜出来的方案天然符合字典序
    for (int c = 1; c <= n; ++c) {
        // 🛡️ 判定当前的直线、主对角线、副对角线是否通畅无冲突
        if (!col[c] && !d1[row - c + n] && !d2[row + c]) {
            
            // 🎯 施法占领当前阵位
            col[c] = d1[row - c + n] = d2[row + c] = true;
            cur.push_back(c);
            
            // 🚀 带着目前的成果，递归推演下一行
            dfs(row + 1);
            
            // 🧹 恢复现场（回溯）：拔出这枚冰后，抹除所有占用痕迹，继续考察下一列
            cur.pop_back();
            col[c] = d1[row - c + n] = d2[row + c] = false;
        }
    }
}

int main() {
    int n_input;
    if (cin >> n_input) {
        n = n_input;
        dfs(0); // 从第 0 行开始摆放冰后
        
        // 📤 打印输出前 3 个符合规则的优秀阵法序列
        for (const auto& sol : firstThree) {
            for (int i = 0; i < n; ++i) {
                if (i > 0) cout << " ";
                cout << sol[i];
            }
            cout << endl;
        }
        // 📤 最后一行为最终结算的总和方案数
        cout << total << endl;
    }
    return 0;
}

🐍 Python 魔法秘籍 (知识点)

  • ✨ 列表的切片复制与备份： 当我们在 Python 中成功发现一个解并想要存入 first_three 时，必须要写成 cur[:]。 这种带有冒号的写法国防级地复制了一份当前的独立数据副本。如果是直接写 cur，由于它是引用传递，后续回溯会把里面的元素全部清空，导致你最后得到的都是空的列表！
  • 🚀 闭包与 nonlocal 关键字： 在内嵌函数 dfs() 里，如果我们想对外面定义的计数器 total 进行修改和累加，必须在开头写一行 nonlocal total，明确告诉 Python 这不是一个局部的全新变量，而是外面那个核心总计数器。

💻 Python 终极法术 (清晰直观回溯版)

import sys

def solve_n_queens(n):
    # 📖 准备冲突记录本，乘 2 是为了给对角线相加相减预留足够的空间
    col = [False] * (n + 1)
    d1 = [False] * (2 * n + 1)  # 主对角线：row - col + n
    d2 = [False] * (2 * n + 1)  # 副对角线：row + col
    
    cur = []
    first_three = []
    total = 0

    def dfs(row):
        nonlocal total # 声明使用闭包外部的全局累计变量
        
        # 🏁 终止条件：所有行都已经成功安排就绪！
        if row == n:
            total += 1
            if len(first_three) < 3:
                first_three.append(cur[:]) # ⚠️ 必须用切片 [:] 制作数据副本备份！
            return
            
        # 🔍 按列号 1 到 n 的升序依次尝试放置冰后
        for c in range(1, n + 1):
            # 🛡️ 判定这三个方位的阵法是否有冲突
            if not col[c] and not d1[row - c + n] and not d2[row + c]:
                
                // 🎯 实施占领，并记录当前列号
                col[c] = d1[row - c + n] = d2[row + c] = True
                cur.append(c)
                
                // 🚀 递归向下摆放下一行
                dfs(row + 1)
                
                // 🧹 清除现场（回溯）：恢复所有的自由通行状态
                cur.pop()
                col[c] = d1[row - c + n] = d2[row + c] = False

    dfs(0) # 从第 0 行正式开始深度搜索之旅
    return first_three, total

def main():
    # 📥 读取输入的棋盘大小限制
    lines = sys.stdin.read().split()
    if not lines:
        return
    n = int(lines[0])
    
    first_three, total = solve_n_queens(n)
    
    # 🏆 格式化打印出前三个合法解
    for sol in first_three:
        print(' '.join(map(str, sol)))
        
    # 🏆 最后一行为全部方案的总结
    print(total)

if __name__ == "__main__":
    main()

💡 技巧和重点详细总结 (回溯阵法核心秘籍)

  • 🌟 回溯（Backtracking）的至高心法： 回溯算法最生动的比喻就是“走迷宫”。当你来到一处岔路口（一列），你先试探着往里走（dfs 深入下一行）。如果走到头发现是死胡同或者已经拿到了正确的路线，你必须沿着原路**倒退回来**。在倒退回岔路口的过程中，你必须把你沿途做的所有标记、放下的路标全部原封不动地**擦除抹净**（恢复占用状态为 false），这样你才能清清白白地去尝试另一条全新的岔路！
  • 🌟 对角线常量化技巧： 这是做所有网格棋盘类冲突题目最精妙的降维打击武器。行号和列号的数学加减组合，能够把纷繁复杂的倾斜对角线完美投影到一条一维的直线索引数组里。不需要用多重循环去沿着斜线数格子供判断，用 $O(1)$ 的时间查一下布尔数组就能立刻知道是否有冲突，这是算法大师的核心提速法宝！
  • 🌟 字典序控制： 很多初学者会在搜出一堆答案后，再写一个排序算法去给答案进行字典序排列，这是非常低效的。只要我们在每一层写 for 循环时，控制列号由小到大递增（1, 2, 3 ... n），那么递归树在生长的过程中，左边的树枝一定比右边的树枝先成熟。这样最先被捕获的三个答案自然而然就是字典序最小的前三个，完全免去了后期的排序开销！
