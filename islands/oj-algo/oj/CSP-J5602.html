📜 任务清单 (题目描述)

每一组骑士都会争夺名次。这些对决中允许出现并列的情况！

例如 2 组骑士 A 和 B，可能会出现：并列第一；A 第一 B 第二；B 第一 A 第二，一共 3 种组合。

计算任务： 请问 n 组骑手最终的名次可能有多少种组合？因为这个数字可能很大，请对 10056 取模（即求余数）。

📥 魔法输入阵：
(第一行是测试次数 T，接下来是每次比赛的组数 n)
3
1
2
3
📤 奇迹输出阵：
(输出格式需要是 Case i: 答案)
Case 1: 1
Case 2: 3
Case 3: 13

🧠 魔法道具箱 (核心图解：名次争夺战)

要算出总组合数，我们可以将问题具体化：如果 n 个人比赛，最终占据了 j 个名次（比如 3 个人只占据了 2 个名次，说明有两个人并列），情况是怎样的？

🛡️ 动态规划：让第 i 个骑士入场 🛡️

假设我们用 f(i, j) 表示 i 个骑士比赛，最终产生了 j 个名次的方案数。

  • 选择 1 选择并列： 第 i 个骑士入场时，他可以和前面的骑士并列！前面的 i-1 人已经占了 j 个名次，所以他有 j 个位置可以选择加入。
    👉 带来的方案数：j * f(i-1, j)
  • 选择 2 独占名次： 第 i 个骑士不愿并列，他自己霸占一个新名次！这意味着前面的人其实只占了 j-1 个名次，而他可以在最终这 j 个名次中挑选任意一个名次插入（比如抢第一，或者落到最后）。
    👉 带来的方案数：j * f(i-1, j-1)
💡 状态转移公式（加起来并提取公因数 j）：
f(i, j) = (f(i-1, j) + f(i-1, j-1)) * j % 10056

最终 n 个骑士的总方案数就是：把他们分出 1 个名次一直到分出 n 个名次的方案数全部加起来！

📘 C++ 魔法秘籍 (知识点)

  • ⚙️ 一次预处理，终生受益 (Precomputation)： 题目中有 T 次查询，如果我们每次都重新算一遍，魔法阵就会过载（Time Limit Exceeded）。所以我们先用两个大循环把所有答案算好存在 ans 数组里，后面查询时直接 O(1) 输出！
  • 🛡️ 步步取模： 不要等最后才对 10056 取模！在乘法和加法的每一步都要取模，防止数字过大导致 C++ 的 int 类型溢出。

💻 C++ 终极法术 (代码详解)

#include <iostream>
using namespace std;

const int N = 1005;
const int M = 10056;
int f[N][N], ans[N] = {0, 1}; // 魔法阵：f 是状态，ans 记录总和，ans 初始化为 1

int main() {
    // 🎨 预处理：每个人与自己并列，只有 1 个名次，方案数为 1
    for (int i = 1; i < N; i++) f[i][1] = 1; 
    
    // ✨ 核心递推：计算所有的名次组合可能
    for (int i = 2; i < N; i++) {
        for (int j = 1; j <= i; j++) {
            // 并列现有名次 + 独占新名次 (注意处处取模)
            f[i][j] = (f[i-1][j] * j + f[i-1][j-1] * j) % M; 
            
            // 累加当前人数下，不同名次个数的方案数
            ans[i] = (ans[i] + f[i][j]) % M; 
        }
    }
    
    int T;
    cin >> T; // 接收询问次数
    for (int i = 1; i <= T; i++) {
        int n;
        cin >> n;
        // 📤 施放法术，按格式输出答案
        cout << "Case " << i << ": " << ans[n] << endl;
    }
    
    return 0;
}

🐍 Python 魔法秘籍 (知识点)

  • ✨ 二维列表生成： 在 Python 中创建一个 N x N 的二维数组，最标准安全的魔法是 [ * N for _ in range(N)]。千万不要用 [ * N] * N，这会导致所有行都关联在一起，改一个全乱套！
  • 🚀 快速读取大法： 使用 sys.stdin.read().split() 能够一次性把所有输入的数据吞下，然后我们挨个取出来用，这在算法比赛中比不停地写 input() 要快得多！

💻 Python 终极法术 (代码详解)

import sys

def main():
    M = 10056
    N = 1005
    
    # 📖 准备魔法笔记本：f[i][j] 表示 i 个人排成 j 个名次的方案数
    f = [[0] * N for _ in range(N)]
    ans = [0] * N
    ans[1] = 1
    
    # 🏁 每个人与自己并列，只有 1 个名次，方案数为 1
    for i in range(1, N):
        f[i][1] = 1
        
    # ✨ 核心递推：计算所有的名次组合可能
    for i in range(2, N):
        for j in range(1, i + 1):
            # (并列现有名次 + 独占新名次) * 选取位置，步步取模
            f[i][j] = (f[i-1][j] * j + f[i-1][j-1] * j) % M
            
            # 累加结果
            ans[i] = (ans[i] + f[i][j]) % M
            
    # 📥 一次性读取所有魔法输入
    input_data = sys.stdin.read().split()
    if not input_data:
        return
    
    T = int(input_data[0]) # 第一个数是询问次数
    for i in range(1, T + 1):
        n = int(input_data[i])
        # 🏆 格式化输出答案
        print(f"Case {i}: {ans[n]}")

if __name__ == "__main__":
    main()

💡 技巧和重点总结 (划重点啦)

  • 🌟 组合数学的转化： 这道题在数学上相当于求「有序集合划分数」（Fubini numbers/Ordered Bell numbers）。我们通过动态规划，巧妙地将排名的逻辑转化为了向已有排名的缝隙里“插空”和“合并”的操作。
  • 🌟 取模运算的分配律： 在编程中，(a + b) % M 等价于 ((a % M) + (b % M)) % M。同理，乘法也适用！所以我们在递推每一步都勇敢地放手去 % M 吧，它能保证答案正确且绝不溢出！
  • 🌟 时空复杂度的权衡： 虽然查询次数可能很多，但总人数最多才 1000 人。预先花 O(N^2) 的时间算好所有的榜单存起来，之后每次查询只需 O(1) 的时间，这叫做“以空间换时间”，是算法大师们的必修课哦！
