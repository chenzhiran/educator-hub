神奇的方括号咒（压缩规则）：

  • 形如 [D子串] 表示把方括号里面的子串连续重复出现 D 次（1 ≤ D ≤ 99）。
  • 嵌套套娃法则： 方括号内部的子串还可以继续包含更多的方括号压缩咒语，最多可以支持 10 层嵌套。
  • 输入数据只包含大写字母、数字、左方括号 '[' 和右方括号 ']'。 解压完成后的最终字符串总长度不超过 20000。
📥 魔法输入阵：
(含有方括号咒的密码线索)
AC[3FUN]
📤 奇迹输出阵：
(完全解压展开后的真实含义)
ACFUNFUNFUN

更多神秘线索提示：

  • 线索二：[2ABC]XYZ[3Q] 展开结果为：ABCABCXYZQQQ
  • 线索三：X[2A[3B]C]Y 展开时先处理内层 [3B] 变成 BBB，得到 [2ABBBC]，再把整段重复 2 次，最后得到：XABBBCABBBCY

🧠 魔法道具箱 (核心图解：递归解压的套娃艺术)

面对这种“大盒子套小盒子”的嵌套结构，最天然完美的破解武器就是递归（Recursion）！我们将使用“从左到右单指针扫描”的递归下降解析策略。

🪆 纯前端网页图解：以密码 X[2A[3B]C]Y 为例的递归展开过程 🪆
第一层解析（当前指针位置：最左端）
遇到普通字母： 遇到 X，直接装进最终结果。
遇到左括号 [： 暂停当前层的组装！读取重复次数 2。指针右移，向下递归调用第二层去解出括号里的秘密。
第二层解析（进入第一个括号内部）
遇到普通字母： 读到 A，直接记下来。
又遇到左括号 [： 再次暂停！读取重复次数 3。指针继续右移，继续递归调用第三层！
第三层解析（最深处的套娃）
遇到普通字母： 读到 B，记下来。
遇到右括号 ]： 这一层的秘密到头了！第三层宣告结束，向第二层返回解压结果："B"。
第二层恢复执行：
接收到第三层传回的 "B"，根据刚才记录的次数 3，将其自我复制变成 "BBB" 并拼接上先前的 A，目前组合成 "ABBB"。
指针继续右移，读到普通字母 C，拼上去变成 "ABBBC"。
遇到右括号 ]： 第二层功德圆满！结束并向第一层返回："ABBBC"。
第一层恢复执行：
接收到第二层传回的 "ABBBC"，根据外面记录的次数 2，将其复制两遍变成 "ABBBCABBBC"。与开头的 X 拼接得到 "XABBBCABBBC"。
指针最后读完末尾的普通字母 Y。整个字符串成功解析完毕！

✨ 递归三大核心逻辑解析

只要我们在代码里设计一个扫描指针变量 i，让它在字符串中一路向右走，遇到的情况可以完美总结为三种状态：

  • 🔸 普通大写字母： 不需要任何特殊处理，直接把它装进我们当前层用来收集结果的容器里，指针 i 前进 1 步。
  • 🔸 遇到左括号 '['： 指针前进 1 步跳过左括号。接着写一个循环把紧随其后的数字完整地读出来，存为重复次数 D。接着，直接呼叫函数自己（进行递归）。递归函数会帮我们把括号深处的东西全部解析好并作为一个整体字符串返回。我们拿到这个返回的子字符串后，把它自我复制叠加 D 次，追加到当前的容器中即可！
  • 🔸 遇到右括号 ']' 或字符串结束： 这代表当前这一层括号的任务圆满完成了！直接退出循环，把当前容器里收集好的字符串返回给呼叫它的上一层。
核心思想：把解压 [D子串] 转化为 “重复 D 次调取 decode() 对子串的解压结果”

📘 C++ 魔法秘籍 (知识点)

  • ⚙️ 传引用控制指针 (&)： 在 C++ 中，为了让所有的递归层级能够共享、并实时修改同一个扫描位置，我们需要将下标变量定义为引用传递：int& i。这样无论在哪一层嵌套里让指针前进了，其他层都能同步感知到最新的扫描进度，绝对不会出现重复读取或漏读的情况。
  • 🛡️ 高效字符串拼接： 在处理高频率的字符串相加时，C++ 的 += 运算符非常高效。由于题目给出了最终解压长度不超过 20000 的限制，直接使用标准的 std::string 收集结果即可轻松通关，无须担心内存过载。

💻 C++ 终极法术 (指针引用递归版)

#include <iostream>
#include <string>
#include <cctype>
using namespace std;

// 🎨 递归解析函数，i 为全局共享的当前解析位置（通过引用传递控制进度）
string decode(const string& s, int& i) {
    string res = ""; // 当前层收集解压片段的容器
    
    // 当还没读到字符串末尾，且没有撞到当前层的右括号时，持续进行解析
    while (i < s.size() && s[i] != ']') {
        if (s[i] == '[') {
            i++; // 👀 成功触发左括号咒，指针前进一步跳过 '['
            
            int d = 0;
            // 🔢 用经典的逐位累加法，读取出紧跟在括号后面的重复次数 D
            while (isdigit(s[i])) {
                d = d * 10 + (s[i] - '0');
                i++;
            }
            
            // 🧙‍♂️ 施展递归套娃分裂术！让下一层函数帮我们把括号里的子串完全解开
            string inner = decode(s, i);
            
            i++; // 👀 递归回来时指针正好停在 ']' 上，前进一步跳过它
            
            // 🔄 按照外仙人指定的次数 d，把解开的子串自我复制叠加到当前结果中
            for (int k = 0; k < d; ++k) {
                res += inner;
            }
        } else {
            // 🔤 遇到的只是普通的魔法字符，直接抓进当前层的容器中，指针前进一步
            res += s[i];
            i++;
        }
    }
    return res; // 返回当前层完全解压拼接好的结果
}

int main() {
    string s;
    // 读取整行外星密码线索
    if (getline(cin, s)) {
        int i = 0; // 从最左端第 0 号位置开启全局解密扫描指针
        cout << decode(s, i) << endl;
    }
    return 0;
}

🐍 Python 魔法秘籍 (知识点)

  • ✨ 多重返回值机制： 由于 Python 的基本变量（如整数）在函数传递时是值传递，无法像 C++ 那样直接修改全局下标，所以我们让递归函数每次同时返回两个宝贝：return ''.join(res), i。这样上一层就能轻松拿到解压好的字符串以及最新的指针位置。
  • 🚀 列表拼接性能优化： 在 Python 中，如果高频执行字符串相加（s = s + char），由于字符串是不可变对象，底层会不断开辟新内存，速度很慢。老练的 Python 程序员会使用列表 res.append() 收集碎片，最后用 ''.join(res) 一次性成型，运行速度直接拉满！

💻 Python 终极法术 (多重返回值递归版)

import sys

# 🎨 递归解析函数：接收原始密码 s 和当前解析起点 i
def decode(s, i):
    """每个递归层级均返回一个元组：(当前层解压后的字符串, 最新推动后的索引位置)"""
    res = [] # 采用列表收集字符碎片，性能更无敌
    n = len(s)
    
    # 当指针没有出界且没有碰到属于当前层结束的右括号时，持续向右推进
    while i < n and s[i] != ']':
        if s[i] == '[':
            i += 1 # 👀 触发左括号，前进一步跳过 '['
            
            d = 0
            # 🔢 连续读取数字字符，拼装出需要重复的倍数 D
            while s[i].isdigit():
                d = d * 10 + int(s[i])
                i += 1
                
            # 🧙‍♂️ 递归调用：把剩下的重任交给下一层套娃，并同步接收对方返回的新指针
            inner, i = decode(s, i)
            
            i += 1  # 👀 下一层结束时指针停留在 ']'，前进一步跳过它
            
            # 🔄 利用 Python 优雅的乘法特性，直接将子串重复 d 次并装进列表
            res.append(inner * d)
        else:
            # 🔤 只是普通的字母，直接追加进当前层的列表，指针前进一步
            res.append(s[i])
            i += 1
            
    # 🏆 把列表里的碎片拼成一个完整的字符串，并连同指针一起奉还给上一层
    return ''.join(res), i

def main():
    # 📥 读取外星密码，剔除两端多余的空格
    s = sys.stdin.read().strip()
    if not s:
        return
        
    # 从 0 号索引位置正式开启大破译之旅
    ans, _ = decode(s, 0)
    
    # 🏆 打印输出完全恢复完毕的星语原文
    print(ans)

if __name__ == "__main__":
    main()

💡 技巧和重点详细总结 (破译者的经验笔记)

  • 🌟 递归下降解析的威力： 这种“用一个指针配合递归函数来解析字符串”的技术，叫做递归下降解析法（Recursive Descent Parsing）。它是计算机编译原理的核心基础！大到解析整个 C++ 语言源程序，小到我们这道解压密码题，只要遇到含有嵌套括号的结构（比如算术表达式、JSON格式、HTML标签），递归下降都是降维打击般的标准解法。
  • 🌟 为什么不需要手动找匹配的右括号？： 初学者做这道题时，最容易走入一个误区：“我看到左括号后，是不是应该先写个循环去往后数，找到对应的右括号在哪里？” 千万不要这样做！ 递归的精髓就在于，每一层函数都只管遇到右括号就闭嘴退出。在上面的代码中，内层的 decode 遇到内层的右括号就会主动刹车，把它完美地处理并返回，外层的循环根本不需要操心内层的结束边界，这就是“各司其职”的递归之美。
  • 🌟 时间复杂度的绝佳表现： 因为整个算法中，全局指针 i 仅仅是从 0 毫无回头路地一路走到了最右端，每个字符只被读取和扫描了刚好一次。唯一的额外花销仅仅是把解压出来的字符拼接到最终结果中。所以整个算法的时间复杂度是 O(最终解压后字符串的长度)。由于题目规定最终长度不超过 20000，程序可以在几毫秒内瞬间完成，效率极其无敌！
