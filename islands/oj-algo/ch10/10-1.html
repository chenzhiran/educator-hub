10.1 DFS 思想与框架

🌲 迷宫探险法则：

假设你在一座极其复杂的黑暗迷宫里，只有一把火把。为了不迷路并且找到出口，你会怎么做？

最稳妥的办法是：选定一个方向，一直往前走，碰到死胡同就退回上一个岔路口，换条路继续走，并在走过的路上做记号。 这种“不撞南墙不回头”的执着探索精神，就是计算机科学中鼎鼎大名的 深度优先搜索 (DFS, Depth First Search)。

一、DFS 的基本概念与递归本质 (10.1.1)

由于 DFS 具有“深入 -> 触底 -> 自动返回”的特性，它简直就是为递归 (Recursion) 量身定制的！

每一次往深处走一步，就像是调用了一次递归函数；每一次退回岔路口，就像是递归函数执行完毕，自动返回到了上一层。只要我们把终点设为“递归的终止条件”，计算机就会自动帮我们在无数条分支中穿梭。

二、DFS 的通用框架与回溯 (10.1.2)

🔥 核心概念：恢复现场 (回溯)

假设你在探险时，拿走了一把金钥匙。当你发现这条路是死胡同并退回来时，你必须把金钥匙放回原处，因为探索另一条路时可能还需要这把钥匙！在代码中，这叫做“撤销操作”或“恢复现场”。不恢复现场，整场搜索都会彻底乱套！

图解 2：DFS 万能魔法框架解析
C++ DFS 通用模板伪代码
// path：记录当前走过的路径 // visited：记录哪些点已经去过 void dfs(int step, vector<int>& path) { // 1. 终止条件 (触底) if (step == 目标深度) { save(path); // 保存这套方案 return; // 自动返回上一层 } // 2. 遍历当前路口的所有选择 for (int i = 0; i < choices.size(); i++) { if (!visited[i]) { // 3. 做选择 (画叉，装入背包) visited[i] = true; path.push_back(choices[i]); // 4. 递归进入下一层深渊！ dfs(step + 1, path); // 5. 撤销选择 (擦掉叉，拿出背包：回溯！) path.pop_back(); visited[i] = false; } } }
Python DFS 通用模板伪代码
# path：记录当前走过的路径 # visited：记录哪些点已经去过 def dfs(step, path): # 1. 终止条件 (触底) if step == 目标深度: save(path.copy()) # 注意Python要存深拷贝 return # 自动返回上一层 # 2. 遍历当前路口的所有选择 for i in range(len(choices)): if not visited[i]: # 3. 做选择 (画叉，装入背包) visited[i] = True path.append(choices[i]) # 4. 递归进入下一层深渊！ dfs(step + 1, path) # 5. 撤销选择 (擦掉叉，拿出背包：回溯！) path.pop() visited[i] = False

三、状态树与搜索过程的可视化理解 (10.1.3)

如果我们把所有的选择都画出来，它会形成一棵巨大的倒长着的树，叫做“状态空间树”。DFS 在这棵树上的探索顺序极其霸道：它总是死死咬住最左边的树枝一路向下，直到树叶才罢休。

四、DFS 的复杂度分析与栈空间影响 (10.1.4)

时间复杂度：看你画出多大的树！
如果树的深度是 N，每个节点有 K 个分支，那么总共要遍历的节点大约是 K 的 N 次方。所以暴力的 DFS 时间复杂度通常是指数级的（O(K的N次方) 或 O(N的阶乘)）。如果是图的遍历，则是 O(顶点数 + 边数)。

空间复杂度：栈的深度
虽然时间可能很久，但 DFS 的空间消耗却非常少！因为它不需要保存所有的节点，它只需要保存“当前正在走”的这条路径。这就引出了一个核心概念：系统调用栈 (Call Stack)。

图解 4：递归背后的“系统调用栈”涨跌

🚨 栈溢出警告 (Stack Overflow)

因为计算机分配给递归的“系统调用栈”内存是很小的。如果你走迷宫一直往下走，走了几万步都没回头（递归太深），栈就会被撑爆，程序直接崩溃！所以 DFS 的空间复杂度为 O(树的深度)，它最怕遇到极度深渊！
