10.4 迷宫寻路

🗺️ 探险家的终极试炼：

在算法世界里，二维网格组成的“迷宫”是 DFS 大显身手最完美的舞台。不论是寻找一条通路，还是计算所有可能的逃生路线，DFS 都能手持火把，在网格的纵横交错间为我们踩出一条通途。

一、二维网格上的 DFS 建模与边界检查 (10.4.1)

坐标的变幻： 在二维网格中，我们用行坐标 r 和列坐标 c 确定一个点。在这个格子上，我们拥有四个可以探索的方向：上、下、左、右。

如果用 4 个 if 语句分别写四种方向移动，代码会变得冗长且丑陋。顶尖的探险家会使用**“方向数组”**的移项魔法，用一个简单的循环搞定四向移动！

二、连通性判断与回溯恢复现场 (10.4.2 & 10.4.3)

走迷宫最怕两件事：一是一脚踩空掉出悬崖（越界），二是一头撞在坚硬的石墙上（障碍物）。因此，每次移动到新格子 (next_r, next_c) 之前，我们都必须经过三道严格的安检：

第一道安检：范围合理 —— 新坐标必须在 0 到最大行数、最大列数的有效棋盘内。

第二道安检：路通无墙 —— 新格子必须是空地，不能是迷宫的障碍墙壁。

第三道安检：拒绝兜圈 —— 新格子必须没有被探险过，也就是 visited 状态不为真。如果在这里兜圈子，程序会陷入死循环。

三、实战：寻找所有路径与路径计数 (10.4.4)

🔥 核心魔法区别：只找一条路 vs 寻找所有路径

1. 如果只找一条逃生路： 只要 check 到了终点，立刻一层一层地返回 true，一路上绝不撤销 visited 标记！因为既然这条路走不通，别人来走也一定走不通。

2. 如果要求数出一共有多少条路径： 当探险队触底拿到答案后，往回走时，必须把脚印擦掉（visited[r][c] = false）。因为换个走法，别的路线可能还会经过这个格子！这就是恢复现场在迷宫中的美妙运用。

图解 3：回溯时擦除脚印（恢复现场）状态状态机
C++ 迷宫所有路径计数实现
#include <iostream> #include <vector> using namespace std; int rows = 4, cols = 4; // 方向数组的魔法：上、下、左、右对应的行和列偏移量 int dr[] = {-1, 1, 0, 0}; int dc[] = {0, 0, -1, 1}; int totalPaths = 0; void dfsHasAllPaths(int r, int c, const vector<vector<int>>& grid, vector<vector<bool>>& visited) { // 1. 终点拦截成功 if (r == rows - 1 && c == cols - 1) { totalPaths++; // 计数加一 return; } // 2. 用循环遍历 4 个方向 for (int i = 0; i < 4; i++) { int nr = r + dr[i]; int nc = c + dc[i]; // 3. 三道严格安检拦截 if (nr >= 0 && nr < rows && nc >= 0 && nc < cols) { // 未越界 if (grid[nr][nc] == 0 && !visited[nr][nc]) { // 无墙且未访问过 visited[nr][nc] = true; // 标记留下脚印 dfsHasAllPaths(nr, nc, grid, visited); // 深入下钻 visited[nr][nc] = false; // 【核心：回溯擦除脚印】 } } } }
Python 迷宫所有路径计数实现
# 方向数组的魔法：上、下、左、右 dr = [-1, 1, 0, 0] dc = [0, 0, -1, 1] rows, cols = 4, 4 total_paths = 0 def dfs_all_paths(r, c, grid, visited): global total_paths # 1. 终点拦截成功 if r == rows - 1 and c == cols - 1: total_paths += 1 return # 2. 遍历四个方向 for i in range(4): nr, nc = r + dr[i], c + dc[i] # 3. 三道严格安检拦截 if 0 <= nr < rows and 0 <= nc < cols: if grid[nr][nc] == 0 and not visited[nr][nc]: visited[nr][nc] = True # 留下脚印 dfs_all_paths(nr, nc, grid, visited) visited[nr][nc] = False # 【核心：回溯擦除脚印】

四、DFS 与 BFS 最短路径的对比 (10.4.5)

下一章我们要学习“广度优先搜索 (BFS)”。它们都是搜索，但在迷宫寻路上的性格南辕北辙：

特性维度 深度优先搜索 (DFS) 🌳 广度优先搜索 (BFS) 🌊
搜索行为 一条道走到黑，不撞南墙不回头。 像水流波纹一样，齐头并进向外扩散。
寻找最短路 非常不擅长。必须把所有路都数一遍，才能挑出最短的。 绝对王者。在无权图迷宫中，第一次踩到终点的路线必定是最短路径！
空间复杂度 极低。只需存当前一根线上的节点。 较高。需要用队列把当前最外圈的所有格子全存下来。
适用场景 连通性判断、数出所有走法路径、数独/棋盘穷举游戏。 寻找迷宫里的最优、最少、最短步数路径。
