10.3 全排列与组合

🌿 经典的穷举试炼：

深度优先搜索最拿手的绝活，就是帮我们把世间万物所有可能的搭配方式**毫无遗漏地穷举出来**。在这一节中，我们将手持探险火把，去征服回溯算法中最核心的四大模块：全排列、去重排列、组合、以及终极挑战 N 皇后问题。

一、无重复元素的全排列 (10.3.1)

核心机制：一个位置一个位置地决定填谁！

比如我们要对 1、2、3 进行全排列。第一层决定第一个空位填谁，第二层决定第二个空位填谁……为了不让同一个数字在一次排列里被重复使用，我们用一个布尔数组 visited 作为标记状态的哨兵。

图解 1：全排列的深度探索与标记流转 (以为例)
C++ 无重复全排列
void dfs(int step, vector<int>& nums, vector<int>& path, vector<bool>& visited) { if (step == nums.size()) { printPath(path); // 收集齐了一组排列 return; } for (int i = 0; i < nums.size(); i++) { if (!visited[i]) { visited[i] = true; path.push_back(nums[i]); // 填空 dfs(step + 1, nums, path, visited); // 深入 path.pop_back(); // 回溯恢复现场 visited[i] = false; } } }
Python 无重复全排列
def dfs(step, nums, path, visited): if step == len(nums): print(path) # 收集齐了一组排列 return for i in range(len(nums)): if not visited[i]: visited[i] = True path.append(nums[i]) # 填空 dfs(step + 1, nums, path, visited) # 深入 path.pop() # 回溯恢复现场 visited[i] = False

二、含重复元素的全排列与去重剪枝 (10.3.2)

痛点： 如果给你的数组是 。由于存在两个相同的 1，如果你按老方法搜，就会产生两组一模一样的 。这叫无效劳作！

💡 智慧的剪枝策略： 先把数组**从小到大排序**，让相同的数字紧紧挨在一起。在同一层选数字时，如果当前数字和上一个数字长得一模一样，且上一个数字刚刚被回溯撤销了标记（说明刚才已经用他作为起点穷举过了），那我们直接把当前这个双胞胎弟弟给剪枝掉！

图解 2：去重剪枝的核心（阻止双胞胎弟弟重复挑大梁）
C++ 带去重全排列
void dfsUnique(int step, vector<int>& nums, vector<int>& path, vector<bool>& visited) { if (step == nums.size()) { print(path); return; } for (int i = 0; i < nums.size(); i++) { if (visited[i]) continue; // 🌟【核心：去重剪枝条件】 // 弟弟和哥哥一样，且哥哥刚刚回溯释放了(visited[i-1]==false) if (i > 0 && nums[i] == nums[i - 1] && !visited[i - 1]) { continue; } visited[i] = true; path.push_back(nums[i]); dfsUnique(step + 1, nums, path, visited); path.pop_back(); visited[i] = false; } }
Python 带去重全排列
def dfs_unique(step, nums, path, visited): if step == len(nums): print(path) return for i in range(len(nums)): if visited[i]: continue # 🌟【核心：去重剪枝条件】 if i > 0 and nums[i] == nums[i - 1] and not visited[i - 1]: continue visited[i] = True path.append(nums[i]) dfs_unique(step + 1, nums, path, visited) path.pop() visited[i] = False

三、组合问题与子集生成的区别 (10.3.3 & 10.3.4)

很多同学容易把“排列”和“组合”搞混。排列讲究顺序（ 和 是不一样的）；而组合不看顺序（ 和 算成同一套方案）。

防止组合重复的黄金定律：传递一个起点指针 startIndex！
每一层挑选数字时，永远只允许挑 startIndex 之后的数字。向后看，绝不回头看！这样就彻底从源头上掐死了镜像重复的可能性。

图解 3：组合与子集生成的树状路线对比
C++ 组合问题写法
// n：数字总数，k：组合定长目标 void combineDFS(int n, int k, int startIndex, vector<int>& path) { // 只有长度达标了才算成功 if (path.size() == k) { print(path); return; } for (int i = startIndex; i <= n; i++) { path.push_back(i); // 🌟【核心：下层起点从 i+1 开始，决不回头】 combineDFS(n, k, i + 1, path); path.pop_back(); // 回溯 } }
Python 组合问题写法
# n：数字总数，k：组合定长目标 def combine_dfs(n, k, start_index, path): # 只有长度达标了才算成功 if len(path) == k: print(path) return for i in range(start_index, n + 1): path.append(i) # 🌟【核心：下层起点从 i+1 开始，决不回头】 combine_dfs(n, k, i + 1, path) path.pop() # 回溯

四、终极一战：N 皇后问题 (10.3.5)

👑 N 皇后的世界名画：

在大小为 N 乘以 N 的国际象棋棋盘上，放置 N 个强大的皇后。要求：任何两个皇后之间都不能处于同一行、同一列、或者同一条正斜线、反斜线上（互相不能残杀）。求一共有多少种摆放方案？

本质就是全排列： 每一行有且只能放一个皇后。问题变成了，第 0 行的皇后放在第几列？第 1 行的皇后放在第几列？这等价于对“列的下标”进行全排列，只是附带了严格的对角线剪枝判定！

图解 4：N 皇后的棋盘格子冲突判断（坐标平移魔法）
C++ N 皇后完整实现
int n = 4; vector<int> queens(n, -1); // queens[row] = col 记录皇后坐标 vector<bool> colMark(n, false), diag1(2*n, false), diag2(2*n, false); void solveNQueens(int row) { if (row == n) { printBoard(queens); return; // 成功安置了所有行 } for (int col = 0; col < n; col++) { // 🌟【核心对角线剪枝】 if (colMark[col] || diag1[row - col + n] || diag2[row + col]) { continue; } // 保护现场 queens[row] = col; colMark[col] = diag1[row - col + n] = diag2[row + col] = true; solveNQueens(row + 1); // 挺进下一行 // 恢复现场 (回溯) colMark[col] = diag1[row - col + n] = diag2[row + col] = false; } }
Python N 皇后完整实现
def solve_n_queens(row, n, queens, col_mark, diag1, diag2): if row == n: print(queens) # 成功安置了所有行 return for col in range(n): # 🌟【核心对角线剪枝】 if col_mark[col] or diag1[row - col + n] or diag2[row + col]: continue # 保护现场 queens[row] = col col_mark[col] = diag1[row - col + n] = diag2[row + col] = True solve_n_queens(row + 1, n, queens, col_mark, diag1, diag2) # 恢复现场 (回溯) col_mark[col] = diag1[row - col + n] = diag2[row + col] = False # 调度启动 n = 4 solve_n_queens(0, n, [-1]*n, [False]*n, [False]*(2*n), [False]*(2*n))
