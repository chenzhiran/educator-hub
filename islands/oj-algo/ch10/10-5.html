10.5 DFS 与 栈

🥞 探险队背包的压缩美学：

在前几节里，我们写 DFS 时只需要疯狂调用函数本身。计算机就像拥有魔力一样，能自动帮我们记住来时的路，并在死胡同里精准退回。这幕后的无名英雄到底是谁？

答案就是——数据结构中的“栈 (Stack)”。这一节，我们将撕开递归的魔法外衣，看看系统是如何用栈实现深度控制的，并学会如何脱离递归，**手动用一个数据栈写出硬核的非递归 DFS 算法**！

一、递归实现背后的系统调用栈 (10.5.1)

隐式的魔法：系统调用栈 (Call Stack)

栈是一个**“后进先出 (LIFO)”**的无底行李箱。每当你调用一次 DFS 函数，系统就会把当前函数的所有参数、局部变量打包成一个“小方块（栈帧）”，狠狠地**压入 (Push)** 系统调用栈的顶部。

当函数触底遇到终止条件 return 时，系统就会把最上面的小方块**弹出 (Pop)** 并销毁，程序自然就回到了上一层函数中断的地方。回溯，本质上就是栈顶元素被弹出释放的过程！

二、手动用栈实现非递归 DFS 的迭代写法 (10.5.2)

🛠️ 脱离递归控制：为什么要手动模拟栈？

我们在 10.1.4 节强调过，系统给予递归调用的深度是很有限的，一旦迷宫深达数万层，程序就会轰然爆出“栈溢出 (Stack Overflow)”错误。为了应对超大规模的数据，硬核算法工程师会选择自己声明一个内存栈数据结构，用 while 循环手动把节点压入、弹出，把控制权牢牢握在自己手里！

手动栈 DFS 的三大核心律动：

1. 起点入栈： 游戏开始，把起点塞进栈里。

2. 弹栈考察： 只要栈没空，就从栈顶抓出一个节点作为当前位置，并标记为已访问。

3. 子节点入栈： 观察当前节点四周有哪些邻居。把**还没去过的邻居**全部狠狠塞进栈顶。这样在下一轮循环里，新塞进去的邻居就会被优先抓出来探索，天然形成深度优先！

图解 2：手动栈在遍历树形结构时的节点进出动线追踪
C++ 手动栈迭代版 DFS
#include <iostream> #include <vector> #include <stack> using namespace std; void dfsIterative(int startNode, const vector<vector<int>>& graph) { // 1. 自己声明一个显式的数据结构栈，避开系统调用栈 stack<int> s; vector<bool> visited(graph.size(), false); s.push(startNode); // 起点入栈 while (!s.empty()) { int curr = s.top(); // 抓取栈顶 s.pop(); // 弹栈 if (visited[curr]) continue; // 查重拦截 visited[curr] = true; cout << curr << " "; // 探索当前节点 // 2. 将没去过的邻居压入栈中 // 提示：为了让遍历顺序和递归一致（先左后右），我们可以反向压入邻居 for (int i = graph[curr].size() - 1; i >= 0; i--) { int neighbor = graph[curr][i]; if (!visited[neighbor]) { s.push(neighbor); // 压入栈顶，下轮循环立刻优先探它 } } } }
Python 手动栈迭代版 DFS
def dfs_iterative(start_node, graph): # Python 中可以直接用普通列表 list 充当栈 # append() 对应压栈，pop() 对应弹栈 stack = [start_node] visited = [False] * len(graph) while stack: curr = stack.pop() # 手动弹出栈顶元素 if visited[curr]: continue # 查重拦截 visited[curr] = True print(curr, end=" ") # 探索当前节点 # 将没去过的邻居反向压入栈中 for neighbor in reversed(graph[curr]): if not visited[neighbor]: stack.append(neighbor) # 压入栈顶

三、递归与迭代两种实现方式的对比与选择 (10.5.3)

掌握了这两种魔法后，在实战中我们到底该挑选哪一个？教研团队为你梳理了它们之间的全面博弈模型：

比较维度 递归版 DFS (系统隐式栈) 🌳 迭代版 DFS (手动显式栈) 🥞
代码书写难度 极度优美简短。只需专注于终止条件和下一层深入，系统为你处理一切。 较为繁琐。需要自己小心维护 while 循环以及入栈弹栈次序。
内存空间安全性 危险。面临系统栈溢出 (Stack Overflow) 的硬伤，**树的深度一般无法跨越万级**。 绝对安全。使用的是堆区内存（你的普通内存条空间），只要电脑内存够大，能承受百万甚至千万深度的搜索。
状态追溯与控制 不直观。状态隐藏在深层的递归树内部，中途强行退回很不方便。 极度直观。整个栈在内存里清清楚楚，你可以随时窥探、清空、或者改变其大小。
终极选择原则 在常规算法题、树与图的浅度遍历、全排列中，**首选递归版**。 如果题目明确说明数据规模巨大、树极深，或者在工业级图形引擎（如迷宫渲染）中，**果断换用手动栈迭代版**。

四、栈在状态保存、路径回溯与深拷贝场景中的应用 (10.5.4)

除了控制搜索的深度，栈在 DFS 中还有一个超级重要的衍生魔法——路径快照与深拷贝 (Deep Copy)。

当我们在网格上深搜时，我们不仅要数出有多少条路径，往往还要把“当前走过的路线”完美地记录下来打印出来。因为栈本身存储了当前从起点到当前节点的所有节点，所以**栈本身就是一条完美的路径快照**！

💡 避坑彩蛋（关于 Python 的浅拷贝陷阱）：

很多同学在用 Python 写回溯记录路径时，喜欢直接写 result.append(path)。千万注意！Python 的列表是引用传递（浅拷贝）。如果直接塞进去，后续 path.pop() 回溯清空时，你已经收集到的结果里的路径也会**跟着一起被清空**，最后只能得到一堆空列表！

正确的做法是利用栈的副本克隆机制，强制进行深层复制：在 C++ 中直接 push 向量（向量会自动复制），在 Python 中必须写成 result.append(path.copy()) 或者 path[:]！
