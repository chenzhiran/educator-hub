9.2 最大值最小化问题

⚖️ 最公平的切蛋糕法则：

在算法竞赛和面试中，你会经常遇到极其绕口的提问：“请把数组分成 M 段，使得这 M 段中，和最大的一段，它的和尽可能地小。”

这句话翻译成大白话就是：我们要干苦力活，老板允许我们分成几个小组，为了不让某一个小组被累死（最大值），我们要尽可能“平均”地分配任务，让最累的那个小组的工作量降到最低（最小化）。

一、单调性特征与判定函数 (9.2.1 & 9.2.2)

遇到“最大值最小化”，千万不要去想怎么切分！我们直接反向思考：

如果我硬性规定：“最累的小组，工作量绝不能超过 Mid！”

  • 如果 Mid 设得特别大（比如 10000），那我们随便怎么分都能满足条件 (Yes)。
  • 如果 Mid 设得特别小（比如 1），那每个人干一点点就达到了上限，老板给的小组数量根本不够用，任务完不成 (No)。

这种随着限制条件逐渐放宽，从“不行”突然变成“行”的特征，就是完美的单调性！我们的目标就是找到这个分界点：第一个出现的 Yes！

二、典型实例：序列分割 (9.2.3 & 9.2.4)

📖 经典题目：抄书问题 (Split Array Largest Sum)

有一排书需要抄写，每本书有不同的页数，顺序不能打乱。例如页数数组是 。你手下有 M = 2 个抄写员。每个抄写员只能抄写连续的几本书。为了让最晚下班的人尽早回家，这 2 个人中，抄写页数最多的那个人，他抄写的页数最少能是多少？

图解 2：Check 函数的贪心判定（假设允许最大工作量 Mid = 15）
C++ 序列分割 (找第一个 Y)
#include <iostream> #include <vector> #include <numeric> // 用于 accumulate 求和 #include <algorithm> using namespace std; // 判断如果每个人最多抄 mid 页，能否用 M 个人抄完？ bool check(long long mid, const vector<int>& books, int M) { int painters = 1; // 至少需要 1 个人 long long currentSum = 0; for (int pages : books) { // 如果遇到一本书比上限还大，直接无法完成 if (pages > mid) return false; if (currentSum + pages > mid) { painters++; // 换下一个人继续抄 currentSum = pages; // 新的人从这本书开始抄 if (painters > M) return false; // 人数超标了 (No) } else { currentSum += pages; // 当前这个人继续抄 } } return true; // 可以完成 (Yes) } long long splitArray(vector<int>& books, int M) { long long left = 0; // 最小可能：最厚的那本书的页数 long long right = 0; // 最大可能：所有书全由一个人抄 for (int pages : books) { left = max(left, (long long)pages); right += pages; } long long ans = right; while (left <= right) { long long mid = left + (right - left) / 2; if (check(mid, books, M)) { ans = mid; // 能完成(Yes)，记在小本本上 right = mid - 1; // 贪心地压缩时间！往左寻找更小的值 } else { left = mid + 1; // 不能完成(No)，时间太紧了，必须放宽限制 } } return ans; }
Python 序列分割 (找第一个 Y)
def check(mid, books, M): painters = 1 current_sum = 0 for pages in books: # 如果遇到一本书比上限还大，直接无法完成 if pages > mid: return False if current_sum + pages > mid: painters += 1 # 换下一个人继续抄 current_sum = pages # 新的人从这本书开始抄 if painters > M: return False # 人数超标了 (No) else: current_sum += pages # 当前这个人继续抄 return True # 可以完成 (Yes) def split_array(books, M): left = max(books) # 最小可能：最厚的那本书的页数 right = sum(books) # 最大可能：所有书全由一个人抄 ans = right while left <= right: mid = left + (right - left) // 2 if check(mid, books, M): ans = mid # 能完成(Yes)，记在小本本上 right = mid - 1 # 贪心地压缩时间！往左边找 else: left = mid + 1 # 不能完成(No)，时间太紧了，必须放宽 return ans # 测试代码: 5 本书，分成 2 个部分 books = [7, 2, 5, 10, 8] print(f"最累的人至少要抄: {split_array(books, 2)} 页") # 输出 18

🎉 总结：解题三把斧

1. 判定单调性：问自己“随着限制的放宽，任务是不是从不可能变成可能？”
2. 写 Check 函数：利用贪心策略，在给定的上限下，模拟分配过程。
3. 套用左半二分模板：遇到 Yes 就记下答案，然后向左（让数值变小）的方向继续死磕！
