9.4 快速幂算法（二分与倍增）

⚡ 从步行到跃迁：

如果要计算 3 的 10 亿次方，普通人会怎么算？把 3 乘上 10 亿次！计算机每秒大概能执行几亿次运算，这可能会花掉整整一秒钟。如果指数变得更大呢？

我们学过二分和倍增，为什么不把指数对半劈开呢？计算出 3 的 5 亿次方，然后再把自己和自己相乘，不就瞬间得到 10 亿次方了吗？这就是快速幂算法 (Fast Exponentiation)的灵魂！

一、指数分解的二分思想 (9.4.1)

二分劈开法： 要求 A 的 10 次方，只要知道 A 的 5 次方，把它自己平方一下就得到了。要求 A 的 5 次方，它是一个奇数，不能完全对半分，那就把它变成： A 的 2 次方的平方，再多乘一个 A！

如此不断把指数对半分，直到指数变成 0。时间复杂度从 O(N) 瞬间降维打击到了 O(log N)。即使指数达到万亿级别，也只需要二三十次运算就能解决！

二、递归实现与二分本质 (9.4.2)

沿着刚才的图解，我们很容易就能写出快速幂的递归版本程序。递归的核心就是找边界：当指数为 0 时，任何非零数的 0 次方都是 1，这就是我们的终点！

C++ 快速幂 (递归二分版)
long long fastPowerRec(long long base, long long exp) { // 1. 终点：指数为 0 时，返回 1 if (exp == 0) return 1; // 2. 二分：交给下一层去算一半的指数 long long half = fastPowerRec(base, exp / 2); // 3. 合并：如果是偶数次方 if (exp % 2 == 0) { return half * half; } // 4. 合并：如果是奇数次方，多乘一个底数自己 else { return half * half * base; } }
Python 快速幂 (递归二分版)
def fast_power_rec(base, exp): # 1. 终点：指数为 0 时，返回 1 if exp == 0: return 1 # 2. 二分：交给下一层去算一半的指数 half = fast_power_rec(base, exp // 2) # 3. 合并：如果是偶数次方 if exp % 2 == 0: return half * half # 4. 合并：如果是奇数次方，多乘一个底数自己 else: return half * half * base

三、非递归实现（倍增写法）与位运算优化 (9.4.3)

💡 为什么要用循环和位运算？

虽然递归版很好理解，但如果遇到极大规模的数据，递归调用会消耗栈空间。在实战中，我们 100% 使用的是基于“二进制拆解（倍增）”的非递归循环版本！并且，我们还可以用位运算把速度压榨到极致！

把指数 11 转换成二进制：1011。它的含义是：11 = 8 + 2 + 1。

也就是说，3 的 11 次方 = 3的8次方 乘 3的2次方 乘 3的1次方。

我们在循环中不断把底数平方（变成它的 2 次方、4 次方、8 次方...），如果发现指数当前的二进制位是 1，就把当前的底数乘进答案里！

四、终极实战：带模运算的快速幂模板 (9.4.4)

由于指数运算的结果往往是个天文数字，轻易就能撑爆甚至 long long。所以算法题通常会要求你“结果对 M 取模（求余数）”。在数学上，乘法可以在每一步运算后都取模，完全不影响最终结果！

C++ 快速幂取模 (位运算终极优化)
long long fastPowerMod(long long base, long long exp, long long mod) { long long result = 1; // 为了安全，先把底数本身也取一次模 base = base % mod; while (exp > 0) { // exp & 1：位运算，判断最后一位是不是 1 (等同于 exp % 2 == 1) if (exp & 1) { result = (result * base) % mod; // 每次乘法后都取模！ } // 底数进行自我翻倍 (倍增的核心！) base = (base * base) % mod; // 每次翻倍后也必须取模！ // exp >>= 1：位运算，将二进制向右移动一位 (等同于 exp / 2) exp >>= 1; } return result; }
Python 快速幂取模 (自带神器与手写版)
# Python 虽然自带了极其优化的 pow(base, exp, mod) 函数 # 但为了学习算法思想，我们依然手写一遍位运算版： def fast_power_mod(base, exp, mod): result = 1 base = base % mod while exp > 0: # exp & 1 判断最后一位是否为 1 if exp & 1: result = (result * base) % mod # 底数平方倍增 base = (base * base) % mod # 位运算向右移一位 exp >>= 1 return result # 测试计算 3 的 10亿 次方，对 1000000007 取模 print(fast_power_mod(3, 1000000000, 1000000007))

🚀 拓展视野：矩阵快速幂

我们今天讲的 base 是一个普通的数字，但你知道吗？如果把 base 换成一个矩阵（二维数组），乘法换成矩阵乘法，这套框架竟然完全不用修改就能跑起来！

这就是传说中的“矩阵快速幂”，它可以用来在 $O(\log N)$ 的极短时间内，求出斐波那契数列的第 10 亿项！算法的通用之美，莫过于此！
