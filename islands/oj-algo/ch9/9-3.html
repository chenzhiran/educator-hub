9.3 最小值最大化问题

🛡️ 木桶效应与最强短板：

上一节我们学习了“最大值最小化（如何让最累的人尽量轻松）”。这一节我们要学习它的镜像兄弟：最小值最大化！

这就像是在扶贫或者分发物资：“请把物资分成 M 份，使得这 M 份中，拿得最少的那个人，他的物资尽可能地多！”

翻译成大白话就是：我们要提高下限（木桶的最短板），让最惨的那个人过得尽量好！

一、单调性反转与模板调整 (9.3.1 & 9.3.2)

单调性的惊天逆转：

如果我们规定：“每个人分到的物资绝对不能少于 Mid！”

  • 如果 Mid 设得特别小（比如 1），那很容易就能满足大家 (Yes)。
  • 如果 Mid 设得特别大（比如 10000），大家胃口都被吊高了，物资根本不够分 (No)。

所以，它的单调性从上一节的 NNNNYYYY 变成了 YYYYNNNN。我们需要寻找的是最后一个 Yes！

二、典型实例 1：最大化最近距离 (牛棚问题) (9.3.3)

🐄 经典重温：愤怒的牛 (Aggressive Cows)

其实我们在第八章已经见过它了，它是“最小值最大化”的绝对代表作。我们要把牛放进隔间，让任意两头牛之间的最小距离，尽可能地大。

判定黑盒： 猜测距离为 mid，如果按 mid 能顺利放下所有的牛（Yes），说明这个距离是可行的。我们要尝试更大的距离（提高下限）！

图解 2：如果猜测 mid = 3（要求间距至少为3），能否放下 3 头牛？
C++ 最大化最近距离
#include <iostream> #include <vector> #include <algorithm> using namespace std; // 判断能否以距离 mid 放下 C 头牛 bool check(int mid, const vector<int>& stalls, int C) { int count = 1; int last_pos = stalls[0]; for (int i = 1; i < stalls.size(); i++) { // 距离达标了才能放下一头牛 if (stalls[i] - last_pos >= mid) { count++; last_pos = stalls[i]; if (count >= C) return true; // 成功 (Yes) } } return false; // 失败 (No) } int maxMinDistance(vector<int>& stalls, int C) { sort(stalls.begin(), stalls.end()); int left = 1; int right = stalls.back() - stalls.front(); int ans = -1; while (left <= right) { int mid = left + (right - left) / 2; if (check(mid, stalls, C)) { ans = mid; // 可以满足，记下答案 left = mid + 1; // 贪心！尝试挑战更大的距离 } else { right = mid - 1; // 距离太大，牛放不下，必须缩短 } } return ans; }
Python 最大化最近距离
def check(mid, stalls, C): count = 1 last_pos = stalls[0] for i in range(1, len(stalls)): # 距离达标了才能放下一头牛 if stalls[i] - last_pos >= mid: count += 1 last_pos = stalls[i] if count >= C: return True # 成功 (Yes) return False # 失败 (No) def max_min_distance(stalls, C): stalls.sort() left = 1 right = stalls[-1] - stalls[0] ans = -1 while left <= right: mid = left + (right - left) // 2 if check(mid, stalls, C): ans = mid # 可以满足，记下答案 left = mid + 1 # 贪心！尝试挑战更大的距离 else: right = mid - 1 # 距离太大放不下，必须缩短 return ans # 测试代码 stalls = [1, 2, 8, 4, 9] C = 3 print(f"最大最近距离: {max_min_distance(stalls, C)}") # 3

三、典型实例 2：分发巧克力 (分配问题变种)

🍫 问题描述：让最惨的人拿到最多

你有很多块不同大小的巧克力，你想把它们分给 K 个小朋友。巧克力的分发必须是连续的子数组（类似于抄书问题）。为了不让任何一个小朋友觉得太委屈，你要保证拿到巧克力总重量最少的那个小朋友，他拿到的重量尽可能地大。

判定黑盒： 猜一个下限 mid，判断：“我能不能让所有 K 个小朋友，每个人拿到的巧克力都至少达到 mid ？”

图解 3：判定能否让所有人的保底达到 Mid = 6
C++ 分发巧克力 (寻找最大的可行解)
// 判断能否分出至少 K 个数组，使得每个数组的和都 >= mid bool check(long long mid, const vector<int>& choco, int K) { int count = 0; long long currentSum = 0; for (int weight : choco) { currentSum += weight; // 一旦这部分达到了下限 mid，就可以分给一个孩子了 if (currentSum >= mid) { count++; currentSum = 0; // 重新开始累积给下一个孩子 } } // 如果能满足的孩子数量 >= K，说明底线 mid 定得还可以，甚至能更高 return count >= K; } long long maximizeMinimum(vector<int>& choco, int K) { long long left = 1; long long right = 0; for (int w : choco) right += w; // 上限是所有巧克力的总和 long long ans = -1; while (left <= right) { long long mid = left + (right - left) / 2; if (check(mid, choco, K)) { ans = mid; // 这个保底线可行 (Yes)！记在小本本上 left = mid + 1; // 贪心！我们要让下限再高一点！往右找 } else { right = mid - 1; // 这个底线太高了，凑不够 K 个人 (No)，必须降低标准 } } return ans; }
Python 分发巧克力 (寻找最大的可行解)
def check(mid, choco, K): count = 0 current_sum = 0 for weight in choco: current_sum += weight # 一旦这部分达到了下限 mid，就可以分给一个孩子了 if current_sum >= mid: count += 1 current_sum = 0 # 重新开始累积 # 如果能满足的孩子数量 >= K，说明底线 mid 定得还算合理 return count >= K def maximize_minimum(choco, K): left = 1 right = sum(choco) ans = -1 while left <= right: mid = left + (right - left) // 2 if check(mid, choco, K): ans = mid # 这个保底线可行 (Yes)！记下答案 left = mid + 1 # 贪心！我们要让下限再高一点！ else: right = mid - 1 # 凑不够人数 (No)，必须降低标准 return ans # 测试代码 choco = [1, 3, 2, 5, 4, 3] K = 3 print(f"最惨的孩子最多能拿到: {maximize_minimum(choco, K)}") # 5

四、二分答案类问题的统一思考流程 (9.3.4)

不论是上一节的最大值最小化（找第一把椅子），还是这节的最小值最大化（找最后一把椅子），二分答案的终极思维流程永远只有三步：

1. 定位目标：识别出题目中类似于“使最大值最小”或“使最小值最大”的反人类句型。把它抽离出来作为 mid。

2. 编写 Check 黑盒：抛开二分，写一个纯粹的、线性的贪心遍历程序，去验证这个 mid 究竟行不行（返回 True 或 False）。

3. 代入 Ans 模板：
- 如果是找最小可行解 (NNNNYYYY)，遇到 True 时 ans = mid; right = mid - 1。
- 如果是找最大可行解 (YYYYNNNN)，遇到 True 时 ans = mid; left = mid + 1。
永远不用担心死循环！
