9.1 数值计算中的二分（解方程）

📈 从离散的台阶，踏入平滑的星轨：

在前面的章节中，我们的二分查找就像在走台阶——不管怎么跳，脚下落的必然是一个整数（数组的下标）。但当我们面对现实世界中的物理问题，或者需要求解一个复杂的数学方程时，答案往往包含无数个小数点。

这时候，普通的“加一、减一”操作就会让我们直接跨过正确答案。欢迎来到实数二分的微观世界！

实数二分的两大黄金法则 (9.1.1 & 9.1.2)：

  1. 绝不加减一： 当发现中点 Mid 太小或者太大时，我们直接让 left = mid 或者 right = mid。因为哪怕是微小的 0.00001，也可能藏着真正的答案！
  2. 固定循环 100 次： 既然 Left 和 Right 永远不可能绝对相等，我们别管什么误差值了，直接写一个执行 100 次的 for 循环！每次区间缩减一半，100 次后区间会缩小到原来的 2 的 100 次方分之一！这精度足以秒杀任何变态算法题，并且绝不死循环！

一、应用一：求算术平方根 (9.1.3)

假设我们不用自带的数学库，要求你自己计算数字 10 的算术平方根，要求精确到小数点后 5 位。我们可以直接猜答案！如果猜的数的平方大于 10，说明猜大了；反之则猜小了。

图解 1：寻找 10 的平方根追踪过程
C++ 计算平方根 (固定 100 次)
#include <iostream> #include <iomanip> // 用于控制输出精度 using namespace std; double findSqrt(double n) { // 处理边界：若 n < 1，右边界需设为 1，否则区间错误 double left = 0.0; double right = max(1.0, n); // 固定循环 100 次，避免任何浮点数精度死循环 for (int i = 0; i < 100; i++) { double mid = left + (right - left) / 2.0; if (mid * mid <= n) { left = mid; // 猜小了，提高下限 (不要加一！) } else { right = mid; // 猜大了，降低上限 (不要减一！) } } return left; // 最后 left 和 right 几乎完全重合 } int main() { // fixed 和 setprecision 保证输出固定 5 位小数 cout << fixed << setprecision(5) << findSqrt(10.0) << endl; return 0; }
Python 计算平方根 (固定 100 次)
def find_sqrt(n): left = 0.0 right = max(1.0, n) # 固定循环 100 次，非常稳健 for _ in range(100): mid = left + (right - left) / 2.0 # 真正的实数除法 / if mid * mid <= n: left = mid # 猜小了，提高下限 (不加一！) else: right = mid # 猜大了，降低上限 (不减一！) return left # 测试：格式化输出保留 5 位小数 print(f"{find_sqrt(10.0):.5f}")

二、应用二：求方程 f(x) = 0 的近似解 (9.1.3)

假设我们有一个复杂且单调递增的方程：f(x) = x的3次方 + x - 15。我们想要找到一个 x，使得 f(x) = 0。

因为方程是单调上升的，如果 f(mid) > 0，说明当前猜的 x 太大了；如果 f(mid) < 0，说明猜小了。原理跟求平方根一模一样！

图解 2：方程寻根的探测追踪过程
C++ 二分解方程
#include <iostream> #include <iomanip> using namespace std; // 1. 定义目标方程 f(x) = x^3 + x - 15 double f(double x) { return x * x * x + x - 15.0; } double solveEquation() { double left = 0.0, right = 10.0; // 2. 固定循环 100 次 for (int i = 0; i < 100; i++) { double mid = left + (right - left) / 2.0; if (f(mid) >= 0) { right = mid; // 结果大于 0，说明 x 猜大了 } else { left = mid; // 结果小于 0，说明 x 猜小了 } } return left; } int main() { cout << "方程的近似根是: " << fixed << setprecision(5) << solveEquation() << endl; return 0; }
Python 二分解方程
def f(x): return x * x * x + x - 15.0 def solve_equation(): left = 0.0 right = 10.0 # 固定循环 100 次，简单又安全 for _ in range(100): mid = left + (right - left) / 2.0 if f(mid) >= 0: right = mid # 猜大了，缩小上限 else: left = mid # 猜小了，提高下限 return left print(f"方程的近似根是: {solve_equation():.5f}")

三、应用三：牛顿迭代法求根 (9.1.4 扩展)

高阶数学中还有一个寻找方程根的极致神器：牛顿迭代法 (Newton's Method)。如果说二分法是个稳扎稳打的老实人，那牛顿法就是一个“顺着切线滑行的狂徒”。

🚀 牛顿法的数学核心：
它不再盲目取中点，而是计算当前点 x 在曲线上的“斜率（导数）”。在当前点画一条切线，这条切线与 X 轴的交点，就是我们下一次探测的位置！

其著名的迭代公式为：
下一个 x = 当前 x 减去 ( 函数值 / 导数值 )

对于方程 f(x) = x的3次方 + x - 15，它的导数（求导法则）是 f'(x) = 3 乘以 x的平方 + 1。

图解 3：牛顿迭代法利用切线进行降维打击
C++ 牛顿迭代法
#include <iostream> #include <iomanip> #include <cmath> using namespace std; // 原方程 f(x) double f(double x) { return x * x * x + x - 15.0; } // 导数方程 f'(x) double derivative_f(double x) { return 3.0 * x * x + 1.0; } double newtonMethod() { double x = 10.0; // 随便猜一个起点 // 只要 f(x) 还不等于 0 (设定一个极小的误差控制) while (abs(f(x)) > 1e-7) { // 核心公式：新 x = 老 x - ( f(x) / f'(x) ) x = x - (f(x) / derivative_f(x)); } return x; } int main() { cout << "牛顿法极速求得近似根: " << fixed << setprecision(5) << newtonMethod() << endl; return 0; }
Python 牛顿迭代法
def f(x): return x * x * x + x - 15.0 def derivative_f(x): return 3.0 * x * x + 1.0 def newton_method(): x = 10.0 # 随便猜一个起点 # abs(f(x)) 趋近于 0，说明找到了根 while abs(f(x)) > 1e-7: # 核心公式：新 x = 老 x - ( f(x) / f'(x) ) x = x - (f(x) / derivative_f(x)) return x # 测试牛顿法 print(f"牛顿法极速求得近似根: {newton_method():.5f}")

🎉 二分法 VS 牛顿法总结

虽然牛顿法非常快，但在算法竞赛和工程运用中，如果遇到了未知的、扭曲的函数，牛顿法有可能会因为切线水平（导数为0）导致程序崩溃，或者在这个坑里反复横跳死循环。

因此，对于单调函数的求根，实数二分（固定循环 100 次）永远是你最可靠、最值得信赖的战友！
