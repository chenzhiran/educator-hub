9.5 ST 表与 LCA 初步

🌲 倍增魔法的究极形态：

在前面的章节中，倍增算法（按 1、2、4、8... 跳跃）展示了极其高效的探路能力。但倍增不仅仅能用来“往前跳”，它还能帮我们预先记录下未来的视野！

这一节，我们将接触两个令初学者闻风丧胆，但实际上非常优雅的高级数据结构：用来“瞬间寻找区间最大值”的 ST 表 (Sparse Table)，以及用来“在树上飞速寻找共同祖先”的 LCA (Lowest Common Ancestor) 算法。

一、ST 表：区间最值的 O(1) 魔法 (9.5.1 & 9.5.2)

痛点： 给你一个长度为 10 万的数组，然后问你 10 万次：“从第 L 个数字到第 R 个数字，最大的是谁？”。如果每次都遍历一遍，程序绝对会超时！

ST 表的核心（预处理的倍增）： 我们创建一个二维数组 f[i][j]，它代表的意思是：从位置 i 开始，连续 2 的 j 次方 个数字中，最大的是谁。

神奇的合并： 长度为 4 的区间，可以劈成两个长度为 2 的区间。所以，f[i] 就等于 左边一半 f[i] 和 右边一半 f[i + 2] 的最大值！我们像盖楼一样，利用极短的时间把所有 2 的指数级区间的最值全部算出来。

图解 1：ST 表的倍增预处理构建

⚡ O(1) 查询的奥秘：重叠没关系！

假设我们要查 之间（长度为 6）的最大值。因为 6 不是 2 的指数，ST 表里没存！怎么办？

找一个刚好比它小的 2 的次方，也就是 4。我们查 （长度4）的最大值，再查 （长度4）的最大值，两者再比一次大小！虽然中间 被查了两次，但这可是求最大值，重复计算完全不影响答案！瞬间给出结果，耗时 O(1)！

图解 2：重叠覆盖查询 (区间长度 6，用两个长度 4 覆盖)
C++ ST 表 (构建与查询)
#include <iostream> #include <vector> #include <cmath> using namespace std; const int MAXN = 100005; const int LOG = 20; // 2的20次方已经超过百万，足够用了 int st[MAXN][LOG]; void buildST(const vector<int>& arr) { int n = arr.size(); // 初始化长度为 1 的区间 for (int i = 0; i < n; i++) { st[i][0] = arr[i]; } // 预处理倍增：外层循环是步长 (j从1到log N) for (int j = 1; (1 << j) <= n; j++) { // 内层循环是起点 i for (int i = 0; i + (1 << j) - 1 < n; i++) { // 长度为 2的j次方 = 两个长度为 2的j-1次方 的区间取最值 st[i][j] = max(st[i][j - 1], st[i + (1 << (j - 1))][j - 1]); } } } int queryST(int L, int R) { // 计算向下取整的 log2(长度) int k = log2(R - L + 1); // O(1) 极致查询：前后两段交叉覆盖取最值！ return max(st[L][k], st[R - (1 << k) + 1][k]); }
Python ST 表 (构建与查询)
import math class SparseTable: def __init__(self, arr): n = len(arr) # log_limit 大约是 log2(N) 的上限 log_limit = int(math.log2(n)) + 1 # 构建二维列表 st[i][j] self.st = [[0] * log_limit for _ in range(n)] # 初始长度为 1 for i in range(n): self.st[i][0] = arr[i] # 倍增预处理 for j in range(1, log_limit): for i in range(n - (1 << j) + 1): self.st[i][j] = max( self.st[i][j - 1], self.st[i + (1 << (j - 1))][j - 1] ) def query(self, L, R): # 找到满足 2^k <= 长度 的最大 k k = int(math.log2(R - L + 1)) # 交叉覆盖取最大值，实现 O(1) 查询 return max( self.st[L][k], self.st[R - (1 << k) + 1][k] )

二、LCA：树上的最近公共祖先 (9.5.3 & 9.5.4)

痛点： 给你一棵极其庞大的家谱树。现在要你找两个亲戚的“最近共同祖先”。如果一步一步往上爬，最坏可能会爬 N 步。

树上倍增法的魔法： 我们和 ST 表一样，预先记录好：每个人往上跳 1 代、2 代、4 代、8 代... 的祖先是谁。（也是 up[u][j] = up[ up[u][j-1] ][j-1] 的组合魔法！）

查询流程：
1. 高度齐平： 如果两人辈分不同，辈分低的人先用倍增大步跳，跳到同一高度。
2. 比翼双飞： 两人一起往上跳！先跳最大的步数（比如 16 代），如果跳到的祖先一样，说明跳过头了，不跳！如果跳到的祖先不一样，说明还得往上找，跳！依次尝试 8 代、4 代、2 代... 最后他们一定会停在最近公共祖先的正下方！

图解 3：LCA 的树上倍增齐飞过程
C++ LCA 倍增查询 (在线查询)
// 假设 up[i][j] 已预处理，表示节点 i 的 2^j 级祖先 // 假设 depth[i] 记录了每个节点的深度 int getLCA(int u, int v) { // 1. 高度齐平：保证 u 的深度大于等于 v if (depth[u] < depth[v]) swap(u, v); // 把 u 提拔到和 v 一样的高度 for (int j = LOG; j >= 0; j--) { if (depth[u] - (1 << j) >= depth[v]) { u = up[u][j]; // 向上跳跃！ } } // 巧了，原来 v 本身就是 u 的祖先！ if (u == v) return u; // 2. 比翼双飞：一起往上跳 for (int j = LOG; j >= 0; j--) { // 如果跳到的祖先不同，就放心大胆地跳 if (up[u][j] != up[v][j]) { u = up[u][j]; v = up[v][j]; } } // 最终 u 和 v 停在公共祖先的正下方 return up[u][0]; // 返回 u 的父亲，也就是最终答案 }
Python LCA 倍增查询
# 假设 up[i][j] 表示节点 i 的 2^j 级祖先 # depth[i] 记录了每个节点的深度 def get_lca(u, v, up, depth, MAX_LOG): # 1. 高度齐平：保证 u 是较深的节点 if depth[u] < depth[v]: u, v = v, u # 把 u 提拔到和 v 一样的高度 for j in range(MAX_LOG, -1, -1): if depth[u] - (1 << j) >= depth[v]: u = up[u][j] # 向上跳跃！ # 如果已经在同一点，它就是 LCA if u == v: return u # 2. 比翼双飞：一起往上跳 for j in range(MAX_LOG, -1, -1): # 祖先不同才跳 (没跳过头) if up[u][j] != up[v][j]: u = up[u][j] v = up[v][j] # 最后停在 LCA 的正下方一层 return up[u][0]

三、小结：ST表与LCA的共同本质 (9.5.5)

💡 终极思考：倍增法为什么这么快？

无论是数组里的 ST 表，还是树上的 LCA，它们的本质完全一样：空间换时间，用预处理建立 2 的指数级“跳板”。

它们都利用了一个极其关键的递推关系：
规模(2的K次方) = 规模(2的K-1次方) + 规模(2的K-1次方)。
正是因为这种层层依赖的组合关系，我们才能用 $O(N \log N)$ 的时间提前把所有的跳板搭好，从而换来在查询时 $O(1)$ 或 $O(\log N)$ 的极致速度！这是算法中“预处理思想”的巅峰之作！
