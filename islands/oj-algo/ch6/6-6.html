6.6 递归与分治策略

⚔️ 折筷子的哲学与蚂蚁吃大象：

一把粗糙的筷子很难折断，但如果把它分成一根一根的，哪怕是小孩子也能轻易折断。一只大象很大，但如果分成一小块一小块，蚂蚁群也能把它搬空。在编程世界中，面对庞大而复杂的数据，最聪明的办法就是 “分而治之” (Divide and Conquer)。

分治策略是算法设计中最伟大的思想之一，而递归，正是实现分治策略最完美的武器！

一、分治思想的三步神功

任何一个标准的分治算法，在它的体内都流淌着“三步走”的血液：

  1. 分解 (Divide)： 挥舞大刀，将原问题拦腰斩断，划分为几个规模较小、但结构与原问题完全相同的子问题。
  2. 解决 (Conquer)： 若子问题规模足够小（比如小到了只剩 1 个数字），就直接出手解决它；若还不够小，就继续召唤“递归”，让它自己解决自己。
  3. 合并 (Combine)： 将所有已经解决的子问题的答案，像拼图一样拼凑起来，形成原问题的最终答案！

二、超酷视觉案例：谢尔宾斯基地毯 (Sierpinski Carpet)

分治法不仅能做枯燥的数学计算，它还是数字艺术的魔法棒！让我们用分治法画一张无穷无尽的“谢尔宾斯基地毯”。

它的规则极度简单，却能画出极其复杂的图形：
将一个正方形切成 3 行 3 列共 9 个小正方形。然后把最中间的那个小正方形挖空（变成白色）。接下来，对剩下的 8 个小正方形，重复这个“切 9 块、挖中间”的动作！

C++ 画地毯 (矩阵操作)
// 递归函数：在 grid 中画出一块左上角为 (r, c)，边长为 size 的地毯 void drawCarpet(int r, int c, int size, vector<string>& grid) { if (size == 1) { // 1. 基准情形：只有1格大小时，画一个实心方块 grid[r][c] = '#'; return; } // 2. 分解：将边长分成 3 份 int sub = size / 3; // 遍历 3 行 3 列 for (int i = 0; i < 3; i++) { for (int j = 0; j < 3; j++) { // 如果是正中间的那一块，我们跳过它（不画，留白） if (i == 1 && j == 1) continue; // 3. 解决：对其余 8 块区域，递归调用自己去画更小的地毯！ drawCarpet(r + i * sub, c + j * sub, sub, grid); } } }
Python 画地毯 (字符串魔法)
# Python 可以利用列表拼接，非常优雅地合并结果 def carpet(n): # 1. 基准情形：边长为 1，直接返回实心块 if n == 1: return ["#"] # 2. 分解与解决：先要小弟帮我画出边长为 n/3 的小地毯 sub_carpet = carpet(n // 3) # 3. 合并：把这 8 个小地毯拼成一个大地毯！ res = [] # 顶部 3 块连在一起 for row in sub_carpet: res.append(row * 3) # 中间 2 块，夹着一大片空白 for row in sub_carpet: res.append(row + " " * (n // 3) + row) # 底部 3 块连在一起 for row in sub_carpet: res.append(row * 3) return res
🪄 代码魔法原理解析：

1. 分解 (Divide)： 不管当前的地毯有多大，我们都把它的边长除以 3 (size / 3 或 n // 3)，这样就把一块大地毯分成了 9 块小地毯的区域。

2. 解决 (Conquer) 与挖空： 我们遍历这 3 行 3 列共 9 个区域。当走到正中间那块（即 i == 1 且 j == 1）时，直接跳过什么都不画，这就实现了“挖空中间”的效果。对于其余的 8 块，我们调用函数自己，继续在这个更小的区域里画地毯。

3. 合并 (Combine)： 在 Python 代码中，我们利用列表推导式，把小地毯的字符串拼接起来。顶部拼接 3 块，中间拼接 2 块并塞入一段空格，底部再拼接 3 块，最终就返回了一张充满神秘感的大地毯！

三、典型战役：快速排序 (Quick Sort) 的粗暴分治

除了画画，在处理数据时，快速排序 将分治策略体现得淋漓尽致。它的套路是：随便挑一个基准数，比它小的全一脚踢到左边，比它大的全一脚踢到右边。这样一分，基准数的位置就彻底固定了！左右两边再各自独立使用快排。

C++ 快速排序 (指针原地操作)
// 辅助函数：负责选取基准，并将数组分区 int partition(vector<int>& arr, int low, int high) { // (此处省略双指针交换分区的具体代码) // 返回基准数最终所处的正确位置下标 } void quickSort(vector<int>& arr, int low, int high) { if (low < high) { // 1. 分解 (Divide)：找基准并分区，pi 是基准的新位置 int pi = partition(arr, low, high); // 2. 解决 (Conquer)：基准已在正确位置，递归排序左右两边 quickSort(arr, low, pi - 1); quickSort(arr, pi + 1, high); // 3. 合并 (Combine)：不需要！原数组已经有序了。 } }
Python 快速排序 (列表推导极简版)
def quick_sort(arr): # 基准情形：长度小于等于1的数组直接返回 if len(arr) <= 1: return arr # 1. 分解：随便挑中间的元素作为基准 (Pivot) pivot = arr[len(arr) // 2] # 把数组分成三派：小于、等于、大于基准 left = [x for x in arr if x < pivot] middle = [x for x in arr if x == pivot] right = [x for x in arr if x > pivot] # 2 & 3. 递归解决左右两派，并直接加法合并返回！ return quick_sort(left) + middle + quick_sort(right)
🪄 快速排序原理解析：

1. 选基准与分区 (Partition)： 在 Python 的极简代码中，我们直接取数组中间的数字作为基准 (pivot)。然后用列表推导式，像滤网一样把比它小的数字过滤出来当左派 (left)，相等的当中间派 (middle)，比它大的当右派 (right)。

2. 递归与合并： 左派和右派的数据现在虽然分开了，但各自内部还是乱序的，怎么办？直接把它们各自丢给 quick_sort() 重新来一遍！等它们各自排好序返回后，我们只需要用 + 号把左、中、右拼起来，整个数组就瞬间有序了！

3. C++ 中的原地操作： 真实的 C++ 快速排序为了节省宝贵的内存，不会像 Python 那样创建全新的数组，而是使用两个聪明的指针在原数组上不断交换元素，这被称为“原地分区 (In-place Partition)”，执行速度如闪电一般！

四、复杂度分析：分治算法有多快？

评估一个递归分治算法快不快，我们不需要去看复杂头晕的数学公式，把它想象成一个老板给打工人分配任务的拔河比赛就行了！

老板（当前函数）接到一个规模为 N 的超大项目。他自己不想干，于是把项目劈成两半，交给两个打工人（子递归）去做。等打工人做完后，老板花时间把两半的结果合并起来。

以归并排序为例，我们来算算整个公司完成项目要多久：

  • 树的层数（深度）： 项目规模每次对半切，N 变成 N/2、N/4... 直到切成 1 为止。切的次数也就是这棵递归树的层数，在数学上这就是 O(log N) 层。
  • 每一层的工作量： 虽然任务被切得越来越碎，但如果你把每一层所有打工人的手头工作加起来，总零件数依然是 N。老板在每一层合并这些零件，需要花费 O(N) 的时间。
  • 总时间复杂度： 每层合并花费 N 的时间，一共有 log N 层。两者相乘，就得出了归并排序的终极速度：O(N log N)！这是基于比较的排序算法中最快的境界！

五、终极辨析：分治与递归是一回事吗？

许多初学者会把这两个词混为一谈。其实，它们的关系就像 “图纸上的战术” 与 “战场上的兵器”：

概念 本质定位 举个通俗的例子
分治法 (Divide & Conquer) 一种宏观的算法设计策略（思想）。它告诉你遇到大问题时“应该怎么做”（切碎了解决）。 军事大将说：“把敌军分割成三块，逐个击破！”这就是分治思想。
递归 (Recursion) 一种具体的代码实现工具（技巧）。函数自己调用自己，用来写出简洁优雅的代码。 士兵拔出佩剑开始砍人。递归是用来执行“逐个击破”战术的完美武器。

💡 总结： 分治法几乎总是用递归来写代码（因为分解出的子问题和原问题结构一样，最适合自我调用）。但递归不仅能用来写分治，还能用来写搜索（如迷宫寻路）、记忆化搜索等！
