6.3 经典问题：汉诺塔

🗼 创世的预言与梵天之塔：

相传在古老的印度，有一座神庙，神庙里有三根底座上的钻石柱子。创世神梵天在创造世界时，在第一根柱子上从下往上按大小顺序摞着 64 片黄金圆盘。神明命令僧侣们将这些圆盘全部移动到第三根柱子上。

预言说，当这 64 个盘子全部移完的那一刻，世界就会在一声霹雳中毁灭。这，就是著名的 汉诺塔 (Tower of Hanoi) 传说！

一、问题描述与规则

在这个游戏中，我们有三根柱子（暂且称它们为 A、B、C），以及 N 个大小不同的盘子。初始状态下，所有盘子都按“大在下、小在上”的顺序套在柱子 A 上。

我们的目标是：把所有的盘子都移动到柱子 C 上。

⚠️ 移动时必须遵守三条铁律：

  • 规则一： 每次只能移动 1 个 盘子。
  • 规则二： 每次移动只能把盘子从一根柱子的最顶端，拿起来放到另一根柱子的最顶端。
  • 规则三： 任何时候，大盘子绝对不能压在小盘子上面！

二、亲身实战：3个盘子的完整搬运过程

为了不被抽象的魔法绕晕，我们先化身为小和尚，亲手把 3个盘子（小号1，中号2，大号3） 从 A 柱完美地搬运到 C 柱。请仔细观察每一步的动态变化！

  1. 第 1 步： 把盘子 1 从 A 移到 C。（因为盘子 1 最小，放哪都不会违规）
  2. 第 2 步： 把盘子 2 从 A 移到 B。
  3. 第 3 步： 把盘子 1 从 C 移到 B。（此时，盘子1和2都成功停靠在 B 柱上了！）
  4. 第 4 步： 关键一步：把最大的盘子 3 从 A 移到 C。（最大的底座终于就位了！）
  5. 第 5 步： 把盘子 1 从 B 移到 A。（给盘子 2 腾出空间）
  6. 第 6 步： 把盘子 2 从 B 移到 C。（盘子 2 完美压在盘子 3 上）
  7. 第 7 步： 把盘子 1 从 A 移到 C。（大功告成！）

三、递归建模：把大象装进冰箱分几步？

仔细回看刚才我们手工搬运 3 个盘子的过程，你有没有发现一个惊人的规律？

前 3 步（盘1移C，盘2移B，盘1移B），其最终结果不就是 把上面的 2 个盘子完整地从 A 柱搬到了 B 柱 吗？

后 3 步（盘1移A，盘2移C，盘1移C），其最终结果不就是 把那 2 个盘子从 B 柱搬到了最终目标 C 柱 吗？

人脑去推演 64 个盘子的每一步会瞬间爆炸。这时候，我们需要召唤 递归的魔法思维：不要盯着底层的细节，要学会“宏观外包”！

💡 宏观三步走战略：假设我们要把 N 个盘子从 A 移到 C。

  1. 腾出底座： 无视最底下的最大盘子，借助 C 作为中转，把上面的一坨 (N-1 个盘子) 全部移动到柱子 B 上。
  2. 移动最大盘： 此时 A 柱只剩最大的盘子，C 柱是空的。我们把最大的盘子直接从 A 移动到 C。
  3. 物归原位： 最后，借助 A 作为中转，把刚才放在 B 柱上的那一坨 (N-1 个盘子) 移动到目标柱子 C 上。

四、递归算法实现与代码图解

看！我们把移动 N 个盘子的大问题，成功拆解成了两个移动 N-1 个盘子的小问题。接下来，只要套用我们在第一节学过的“递归三要素”：

  • 基准情形： 当 n == 1 时，直接把这唯一的一个盘子从起点拿到终点。
  • 递归情形： 按照上面的三步走战略，调用函数自己本身！
C++ 汉诺塔实现
#include <iostream> using namespace std; // 参数：盘子数量n，起点柱，终点柱，辅助柱 void hanoi(int n, char source, char target, char aux) { // 1. 基准情形：只有1个盘子，直接移 if (n == 1) { cout << "将盘子 1 从 " << source << " 移动到 " << target << endl; return; } // 2. 递归情形（三步走战略） // 第一步：将上面 n-1 个盘子从 source 移到 aux，以 target 为辅助 hanoi(n - 1, source, aux, target); // 第二步：将最大的盘子 n 从 source 移到 target cout << "将盘子 " << n << " 从 " << source << " 移动到 " << target << endl; // 第三步：将刚才放到 aux 上的 n-1 个盘子移到 target，以 source 为辅助 hanoi(n - 1, aux, target, source); } int main() { // 移动 3 个盘子：从 A 移动到 C，借助 B hanoi(3, 'A', 'C', 'B'); return 0; }

五、步数递推与复杂度证明

僧侣们要把 64 个盘子全部移完，到底需要多久呢？让我们来算一笔账。假设移动 N 个盘子需要的最少总步数为 H(N)。

根据我们的“三步走”战略，完成一次完整的搬运需要：

1. 把 N-1 个盘子移走（耗费 H(N-1) 步）

2. 把最大的盘子移过去（耗费 1 步）

3. 把 N-1 个盘子移回来（又耗费 H(N-1) 步）

得出递推公式：H(N) = 2 * H(N-1) + 1

通过数学上的等比数列求和可以证明，它的通项公式为：H(N) = 2N - 1（2的N次方减1）。这就是经典的 指数级复杂度 O(2N)！

六、延伸：四柱汉诺塔的降维打击

如果神明稍微仁慈一点，给了僧侣们 4根柱子（A、B、C、D）呢？

仅仅增加了一根辅助柱，算法的复杂度就会发生断崖式的下降！这就是著名的 Frame-Stewart 算法。在四根柱子的情况下，我们不再是只能把上面的盘子捆绑成“一坨”，而是可以把它们分成“两坨”。

💡 四柱宏观战略（假设有 N 个盘子，我们从中选择一个最佳的分界点 K）：
  1. 第一坨 (K个盘子)： 使用 4 根柱子，将顶部的 K 个盘子从 A 移动到某个辅助柱（比如 B）上停放。
  2. 第二坨 (N-K个盘子)： 此时 B 柱已经被霸占。我们只能使用剩下的 3 根柱子（A、C、D），将底层剩余的 N-K 个盘子从 A 移动到目标柱 D。
  3. 合并： 再次使用 4 根柱子，将刚才停放在 B 柱上的 K 个盘子移动到目标柱 D。

通过动态规划寻找最完美的 K 值，64 个盘子的搬运步数将从恐怖的 1.84 * 1019 步 锐减到仅仅 18433 步！这就是算法与数学结合所产生的“降维打击”奇迹。
