6.2 递归执行过程与栈帧

🥞 煎饼果子与魔法电梯：

你知道电脑是怎么记住“自己调用自己”走到哪一步的吗？想象你在食堂拿盘子，新洗好的盘子总是摞在最上面，而你拿盘子时也只能从最上面拿。这叫 后进先出 (LIFO)。

在编程魔法中，这个装盘子的容器叫做 “函数调用栈 (Call Stack)”，而每一个盘子，就是一个 “栈帧 (Stack Frame)”！

一、函数调用栈与栈帧结构

当一个函数开始执行时，电脑会在内存的“栈”区为它分配一个专属的“工作隔间”，这就是栈帧。它就像一张便签纸，记录着当前函数的所有私密信息。如果这个函数又调用了别的函数（或者它自己），电脑就会在上面再叠一张新的便签纸。

🔍 栈帧里到底装了什么？

  • 局部变量： 函数内部自己定义的变量（比如 int a = 5;），别的函数看不见。
  • 参数： 从外面传进来的数据（比如 sum(3) 里的 3）。
  • 返回地址： 这个最重要！它记录了“等我执行完，我该把结果交给谁？我该回到代码的哪一行继续执行？”

二、压栈与弹栈的魔法秀

让我们用一个简单的例子：计算 1 到 n 的和（例如 sum(3) = 3 + 2 + 1 = 6），来亲眼看看递归是如何一层层压入，又一层层弹出的。

C++ 累加求和
int sum(int n) { // 基准情形：当 n 为 1 时返回 1 if (n == 1) { return 1; } // 递归情形：自己加上更小规模的和 return n + sum(n - 1); } int main() { int result = sum(3); return 0; }
Python 累加求和
def sum_n(n): # 基准情形：当 n 为 1 时返回 1 if n == 1: return 1 # 递归情形：自己加上更小规模的和 return n + sum_n(n - 1) # 主程序调用 result = sum_n(3)

三、危险边缘：递归深度与栈溢出

电脑的内存是有限的，分配给“函数调用栈”的空间就更小了（通常只有几兆字节，MB）。如果你忘记写“基准情形”，或者问题规模太大（比如调用 sum(100000)），就会导致盘子越摞越高，最后把天花板顶破！这就叫 栈溢出 (Stack Overflow)，程序会当场崩溃闪退。

🚨 致命错误演示：忘记基准情形

下面的代码就像一个刹车失灵的汽车，会疯狂地压栈，直到系统强制将它杀死。

C++ 崩溃代码
void endless_loop() { // 致命错误：没有写 if(基准条件) return; // 函数直接调用自己，无休无止！ endless_loop(); } int main() { endless_loop(); // 砰！程序崩溃 return 0; }
Python 崩溃代码
def endless_loop(): # 致命错误：没有写 if 基准条件: return # 函数直接调用自己，无休无止！ endless_loop() # 主程序调用 endless_loop() # Python 会抛出 RecursionError

四、火眼金睛：通过打印观察执行过程

觉得脑内模拟压栈弹栈太抽象了？我们可以通过一个小技巧让程序自己“说”出它的执行过程。利用一个表示深度的参数，打印不同数量的空格，就能画出一棵漂亮的“递归树”！

C++ 打印执行缩进
#include <iostream> #include <string> using namespace std; void countdown(int n, string indent) { cout << indent << "进入 countdown(" << n << ")" << endl; if (n == 0) { cout << indent << " 触底！准备返回" << endl; return; } // 递归调用，缩进增加 4 个空格 countdown(n - 1, indent + " "); cout << indent << "退出 countdown(" << n << ")" << endl; } int main() { countdown(2, ""); return 0; }
Python 打印执行缩进
def countdown(n, depth=0): # 深度乘空格，生成缩进字符串 indent = " " * depth print(f"{indent}进入 countdown({n})") if n == 0: print(f"{indent} 触底！准备返回") return # 递归调用，深度加 1 countdown(n - 1, depth + 1) print(f"{indent}退出 countdown({n})") # 主程序调用 countdown(2)
