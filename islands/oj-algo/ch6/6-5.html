6.5 记忆化搜索：魔法备忘录

🧠 不做无用功的智慧：

想象你在玩一个极其复杂的迷宫游戏，每次走到一个分叉口，你都要花好几个小时去探路。如果某天你又走到了同一个路口，你会傻傻地再探一遍路吗？

当然不会！聪明人会掏出一个小本本（备忘录），把探过的路口答案记下来。下次遇到直接翻开本子抄答案！这就是把普通递归升级为“神级递归”的法宝——记忆化搜索 (Memoization)。

一、重叠子问题与重复计算的深渊

在没有“备忘录”的情况下，纯粹的递归会像一个失忆的人，疯狂地重复计算已经被解开的谜题。这在计算机科学中被称为 重叠子问题。

二、备忘录的构建与查询：记忆化三步走

为了拯救“失忆”的递归，我们要在它的内部加装一个“记忆库”。这也被称为自顶向下动态规划。具体怎么做呢？

  1. 创建备忘录： 建立一个全局的数组或哈希表（比如 memo[]），用来记录每个参数对应的答案。初始时，把里面填满代表“没算过”的特殊值（通常填 -1 或 None）。
  2. 进来先查表： 在递归函数的开头（基准条件之后），第一时间去备忘录里查一下：if memo[n] != -1，如果有答案，就直接 return memo[n]，瞬间结束战斗！
  3. 算完快记下： 如果备忘录里没有，那就老老实实往下算。但千万别忘了，在 return 答案给上一层之前，一定要把刚算出的心血记在小本本上：memo[n] = 刚算出的结果。

三、实战：斐波那契数列的记忆化

让我们用代码给“失忆”的斐波那契吃下一颗“记忆胶囊”。

C++ 记忆化搜索
#include <iostream> #include <vector> using namespace std; // 1. 建立备忘录，初始化全为 -1，表示没算过 vector<long long> memo(100, -1); long long fib_memo(int n) { if (n <= 2) return 1; // 2. 进来先查表！如果有，直接抄答案返回 if (memo[n] != -1) { return memo[n]; } // 3. 如果表里没有，老实计算... long long res = fib_memo(n - 1) + fib_memo(n - 2); // 4. 算完别忘了！记入备忘录后再返回 memo[n] = res; return res; }
Python 记忆化搜索
# 1. 建立备忘录，可以使用字典(哈希表) memo = {} def fib_memo(n): if n <= 2: return 1 # 2. 进来先查表！如果有，直接抄答案返回 if n in memo: return memo[n] # 3. 如果表里没有，老实计算... res = fib_memo(n - 1) + fib_memo(n - 2) # 4. 算完别忘了！记入备忘录后再返回 memo[n] = res return res # Python还有个一键魔法：@cache 装饰器 # 导入 from functools import cache 后， # 直接在函数上面写 @cache 即可自动实现记忆化！

四、进阶示例：网格路径探险

问题： 机器人从一个 m * n 网格的左上角 (0, 0) 出发，每次只能向右或向下移动一步，到达右下角终点有多少种走法？

重叠子问题分析： 机器人走到坐标 (1, 1) 可以是“先右后下”，也可以是“先下后右”。不管怎么走来的，从 (1, 1) 继续走到终点的方案数是固定不变的！如果不用备忘录，机器人每次经过 (1, 1) 都会把后续的地图重新探一遍。

C++ 网格路径二维记忆化
// 二维备忘录 memo[m][n]，初始化填入 -1 int uniquePaths(int r, int c, int m, int n, vector<vector<int>>& memo) { // 越界处理 if (r >= m || c >= n) return 0; // 基准情形：到达终点算作 1 种路径 if (r == m - 1 && c == n - 1) return 1; // 进来先查表 if (memo[r][c] != -1) { return memo[r][c]; } // 计算：向右走的方案数 + 向下走的方案数 int paths = uniquePaths(r, c + 1, m, n, memo) + uniquePaths(r + 1, c, m, n, memo); // 算完快记下 memo[r][c] = paths; return paths; }
Python 网格路径二维记忆化
def uniquePaths(r, c, m, n, memo): # 越界处理 if r >= m or c >= n: return 0 # 基准情形：到达终点算作 1 种路径 if r == m - 1 and c == n - 1: return 1 # 进来先查表 (元组作为字典的键) if (r, c) in memo: return memo[(r, c)] # 计算：向右走的方案数 + 向下走的方案数 paths = uniquePaths(r, c + 1, m, n, memo) + \ uniquePaths(r + 1, c, m, n, memo) # 算完快记下 memo[(r, c)] = paths return paths

五、高阶挑战：背包问题的记忆化

问题： 给定 N 个物品，每个物品有重量 w[i] 和价值 v[i]。你的背包最大承重为 W，在不超过容量的前提下，如何挑选物品使得总价值最大？

在这个问题里，决定你当前状态的因素有两个：1. 当前正在看第几个物品；2. 背包还剩多少容量。所以，我们的备忘录也必须是二维的：memo[index][capacity]。

C++ 背包问题 (记忆化)
// memo[index][capacity] 初始化为 -1 int knapsack(int i, int w, vector<int>& wt, vector<int>& val, vector<vector<int>>& memo) { // 基准情形：物品选完，或容量耗尽 if (i == wt.size() || w == 0) return 0; // 进来先查表 if (memo[i][w] != -1) return memo[i][w]; // 情况1：背包放不下当前物品，只能跳过 if (wt[i] > w) { memo[i][w] = knapsack(i + 1, w, wt, val, memo); } else { // 情况2：能放下，计算【不选】和【选】这两种路线的最大值 int skip = knapsack(i + 1, w, wt, val, memo); int take = val[i] + knapsack(i + 1, w - wt[i], wt, val, memo); memo[i][w] = max(skip, take); } return memo[i][w]; }
Python 背包问题 (记忆化)
def knapsack(i, w, wt, val, memo): # 基准情形：物品选完，或容量耗尽 if i == len(wt) or w == 0: return 0 # 进来先查表 if (i, w) in memo: return memo[(i, w)] # 情况1：背包放不下当前物品，只能跳过 if wt[i] > w: ans = knapsack(i + 1, w, wt, val, memo) else: # 情况2：能放下，计算【不选】和【选】这两种路线的最大值 skip = knapsack(i + 1, w, wt, val, memo) take = val[i] + knapsack(i + 1, w - wt[i], wt, val, memo) ans = max(skip, take) # 算完快记下 memo[(i, w)] = ans return ans

六、终极辨析：记忆化搜索 vs 动态规划

记忆化搜索（Memoization）和动态规划（Dynamic Programming, DP，又叫递推）是解决同一个重叠子问题的两种不同套路。

特性 记忆化搜索 (自顶向下) 动态规划 DP (自底向上)
计算顺序 从目标往下拆解，缺什么算什么 (按需计算) 从地基往上盖，不管后面用不用得着全算一遍
实现方式 递归函数 + 数组/哈希表 多重 for 循环 + 数组
优点 符合人类直觉，好想代码好写，有些无效状态自动跳过 速度极快，没有栈空间消耗，容易进行“滚动数组”的空间优化
缺点 存在递归调用的额外开销，层数极深时可能爆栈 需要严格定义填表顺序，状态转移稍显抽象

🌟 大师的建议：
在算法竞赛或工程实战中，如果你能一眼看穿状态转移的顺序（比如一维数组从左往右填），首选 循环递推(DP)。如果问题像树状分支一样跳跃，状态极其复杂不好确定填表顺序时，果断掏出 记忆化搜索！
