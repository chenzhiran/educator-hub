6.4 递归转递推

🔄 盖楼的两种思维：从屋顶倒推 vs 从地基打起

前面我们学习的“递归”，就像是一个想要直接触碰屋顶的人，他发现够不着，于是问下一层楼怎么盖，下一层再问下下一层……直到问到地基（基准情形），然后再一层层把结果返回来。这叫 自顶向下。

但现实中，聪明的工匠盖楼，一定是先打好地基（已知条件），然后一层一层稳扎稳打地往上盖，直到盖到屋顶。这种思维，就是我们今天要施展的高阶魔法：递推（自底向上）！

一、递推与递归的关系

递归和递推，其实是解决同一个问题的“双胞胎”。它们的数学公式往往是一模一样的，唯一的区别在于计算的方向与存储方式。

递归利用了操作系统的“函数调用栈”来隐式地保存过程，而递推则是我们显式地创建一个“数组”来记录中间答案。因此，递推完美地避开了爆栈的风险，并且省去了反复调用函数带来的时间损耗，可以说是将速度拉满的终极形态！

二、从递归推导递推公式：三步走心法

要把一个已经写好的递归程序，改写成运行速度极快、永不爆栈的递推循环，只需牢记以下“转换三步曲”：

  1. 找准“小账本”（状态定义）： 定义一个数组（比如 dp[i]），用来明确记录每一步的答案。这是递推中最难也最核心的一步，必须想清楚 dp[i] 到底代表什么含义。比如在斐波那契中，它就代表第 i 个斐波那契数。
  2. 垫好“地基”（边界条件）： 递归里的“基准情形（if n == 1 return 1）”，直接变成数组的初始赋值（比如 dp = 1）。因为递推是从底层开始的，如果没有地基，公式就成了无源之水。
  3. 写好“发动机”（递推方程与循环）： 这是魔法自动运转的时刻。递归里的自我调用 f(n) = f(n-1) + ...，在递推中直接变成对已知数组元素的查表相加 dp[i] = dp[i-1] + ...。用一个 for 循环，让 i 从已知地基一路跑向目标 n 即可。

三、实战示例 1：斐波那契数列

我们在第一节中提过，斐波那契数列如果是纯递归，会因为疯狂地重复计算导致电脑卡死。现在我们把它转为递推（自底向上循环计算），不仅不会卡，而且在一瞬间就能给出答案！

C++ 斐波那契 (递推)
long long fib_iterative(int n) { if (n <= 2) return 1; // 1. 准备小账本 (数组) long long dp[n + 1]; // 2. 垫好地基 (边界条件) dp[1] = 1; dp[2] = 1; // 3. 发动机：自底向上循环推导 for (int i = 3; i <= n; i++) { // 查表相加，完全没有函数调用的开销 dp[i] = dp[i - 1] + dp[i - 2]; } return dp[n]; // 直接返回目标结果 }
Python 斐波那契 (递推)
def fib_iterative(n): if n <= 2: return 1 # 1. 准备小账本 (列表) dp = [0] * (n + 1) # 2. 垫好地基 (边界条件) dp[1] = 1 dp[2] = 1 # 3. 发动机：自底向上循环推导 for i in range(3, n + 1): # 查表相加，完全没有函数调用的开销 dp[i] = dp[i - 1] + dp[i - 2] return dp[n] # 直接返回目标结果

四、实战示例 2：爬楼梯问题

问题描述： 假设你正在爬一个 N 阶的楼梯。每次你可以选择跨 1 个台阶或者 2 个台阶。请问，爬到第 N 阶楼梯，一共有多少种不同的走法？

思考逻辑： 我们把“物理上的台阶”抽象为“数学上的状态”。想想最后一步你是怎么到达目标第 N 阶的？
由于每次只能跨 1 步或 2 步，所以你到达第 N 阶之前的那一瞬间，要么站在第 N-1 阶，要么站在第 N-2 阶。
所以：到达第N阶的走法 = 到达第N-1阶的走法 + 到达第N-2阶的走法。这说明爬楼梯本质上也是一个斐波那契的变形体！

C++ 爬楼梯 (递推)
int climbStairs(int n) { if (n == 1) return 1; // 1. 准备小账本 int dp[n + 1]; // 2. 地基：1阶有1种走法，2阶有2种走法(1+1 或 2) dp[1] = 1; dp[2] = 2; // 3. 自底向上推导每一阶的走法 for (int i = 3; i <= n; i++) { dp[i] = dp[i - 1] + dp[i - 2]; } return dp[n]; }
Python 爬楼梯 (递推)
def climbStairs(n): if n == 1: return 1 # 1. 准备小账本 dp = [0] * (n + 1) # 2. 地基：1阶有1种，2阶有2种(1+1 或 2) dp[1] = 1 dp[2] = 2 # 3. 自底向上推导每一阶的走法 for i in range(3, n + 1): dp[i] = dp[i - 1] + dp[i - 2] return dp[n]

五、递归转递推的终极对决与应用场景

对比维度 递归 (自顶向下) 递推 (自底向上)
执行速度 较慢（频繁的函数压栈、弹栈开销） 极快（普通的循环遍历，毫无额外开销）
内存消耗 高（需要极深的函数调用栈，容易爆栈） 极低（只需要一个数组，甚至能优化为几个变量）
代码易读性 优雅直观（公式原样翻译，数学美感极佳） 稍显平淡（需要理清推导顺序和状态定义）
适用场景 问题具有明显树状分支、回溯搜索、不知深度时 绝大多数数列求值、动态规划问题

💡 魔法终极进化：空间复杂度优化到 O(1)

如果你仔细观察斐波那契数列和爬楼梯的递推公式：dp[i] = dp[i-1] + dp[i-2]。你会发现，虽然我们开了一个很长的数组记录历史记录，但实际上，每次计算新楼层时，我们只需要知道紧挨着的前面两个楼层的数据，更早的楼层数据已经被彻底抛弃，再也用不上了！

因此，我们可以连数组都不开，只用三个简单的变量 a（代表 i-2 的值）、b（代表 i-1 的值）、c（代表刚算出来的 i 的值）不断滚动向前即可。

具体来说，就是在每次循环中，用 c = a + b 算出新台阶，然后立刻让位置整体前移：a = b，b = c。旧的数据被无情覆盖，内存占用从 O(N) 瞬间变成 O(1)。这被称为“滚动数组”优化，是递推算法在面试与竞赛中的进阶杀手锏！
