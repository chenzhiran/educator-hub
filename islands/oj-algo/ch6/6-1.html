6.1 递归的基本要素

🪆 童谣与俄罗斯套娃：

“从前有座山，山里有座庙，庙里有个老和尚在讲故事。讲的什么故事呢？从前有座山...”

如果你听过这个故事，或者玩过一层套一层、越套越小的“俄罗斯套娃”，那么恭喜你，你已经领悟了递归（Recursion）的灵魂！

一、递归的定义与思想

在编程世界中，递归 就像一面照着镜子的镜子。它的本质是：一个函数在自己的内部，调用它自己。

面对一个极其庞大复杂的问题，递归思想会教导我们：“不要试图一口吃成个胖子！” 而是把这个大问题，剥洋葱般拆解成一个与原问题“长得一模一样，但规模更小”的子问题。交给“另一个自己”去解决。

二、递归三要素：灵魂的拷问

一个完美的递归魔法，绝不能是“无限死循环”。在编写任何递归函数之前，你必须在脑海中想清楚以下三个关键要素：

1. 基准情形 (Base Case) —— “什么时候停下？”

这是递归的终止条件。就像老和尚的故事总得有个结局，潜水总得踩到海底。基准情形通常是最简单、已知答案的问题规模（比如 n=1 或 n=0）。没有它，程序会直接崩溃（栈溢出）！

2. 递归情形 (Recursive Case) —— “公式是什么？”

这是把“大问题”转化为“小问题”的核心逻辑。你要假设那个规模更小的“自己”已经完美地完成了任务，然后在这个基础上加上当前层的操作，得出最终结果。

3. 向基准情形收敛 (Convergence) —— “方向对吗？”

你每次调用“自己”时，传递进去的参数必须是在缩小问题规模，一步步逼近那个基准情形（海底）。如果参数不变化，或者越变越大，那永远也无法结束！

三、递归函数的结构设计

掌握了三要素，我们就可以总结出所有递归函数的“万能骨架结构”了：

返回值类型 函数名(参数) {
    // 1. 拦截器：最先写基准情形！
    if (满足基准情形) {
        return 直接给出的答案;
    }
    
    // 2. 缩小问题规模，调用自身 (收敛)
    子问题答案 = 函数名(被缩小/修改的参数);
    
    // 3. 递归情形：基于子答案加工，并返回
    return 根据子问题答案计算出当前层的答案;
}

四、经典示例拆解

示例一：阶乘计算 (n!)

数学中，5! = 5 × 4 × 3 × 2 × 1。换个思路看，其实是 5! = 5 × 4!。
由此得出递推公式：f(n) = n × f(n-1)。
它的基准条件是：当 n=1 或 n=0 时，阶乘为 1。

C++ 计算阶乘
#include <iostream> using namespace std; long long factorial(int n) { // 要素 1: 基准情形 if (n == 0 || n == 1) { return 1; } // 要素 2 & 3: 递归情形与收敛 (n-1) return n * factorial(n - 1); } int main() { cout << factorial(5) << endl; // 输出 120 return 0; }
Python 计算阶乘
def factorial(n): # 要素 1: 基准情形 if n == 0 or n == 1: return 1 # 要素 2 & 3: 递归情形与收敛 (n-1) return n * factorial(n - 1) # 测试代码 print(factorial(5)) # 输出 120

示例二：斐波那契数列 (Fibonacci)

斐波那契数列是：1, 1, 2, 3, 5, 8... 从第三项开始，每一项都等于前两项之和。
递归公式：F(n) = F(n-1) + F(n-2)。
这里有两个分支在向下收敛，同时基准情形也变成了两个（第一项和第二项均为1）。

C++ 斐波那契数列
long long fibonacci(int n) { // 基准情形：数列的前两项都是1 if (n == 1 || n == 2) { return 1; } // 递归情形：一分为二，向下收敛 // 注意：这会生成一棵庞大的递归树！ return fibonacci(n - 1) + fibonacci(n - 2); }
Python 斐波那契数列
def fibonacci(n): # 基准情形：数列的前两项都是1 if n == 1 or n == 2: return 1 # 递归情形：一分为二，向下收敛 # 注意：这会生成一棵庞大的递归树！ return fibonacci(n - 1) + fibonacci(n - 2)
